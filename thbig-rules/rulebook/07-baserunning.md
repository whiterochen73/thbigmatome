## 7.1 概要

走塁処理は打席処理（§5）の後続として発生するか、または打席外（盗塁・エンドラン）で独立して発生する。

| 処理種別 | 発生タイミング | 本章のセクション |
|---------|--------------|----------------|
| 積極走塁（走塁表判断） | 打席処理の後 | 7.2 |
| 盗塁 | 打席前（投球前に完結） | 7.3 |
| バント | 打席内（通常打席と置き換え） | 7.4 |
| エンドラン | 打席内（K時に後続処理あり） | 7.5 |
| 前進守備 | 守備側の宣言（打席ごと） | 7.6 |

## 7.2 走塁表（積極走塁）

### 7.2.1 走塁表の概要

外野安打・二塁打・外野フライの後、攻撃側が「より先の塁を狙うか」を判断し、走塁表で成否を判定する。

**判定方法**:
- ダイス: 1d20
- **セーフ（進塁成功）**: ダイス目 ≤ 表の閾値
- **アウト（タッチアウト）**: ダイス目 > 表の閾値
- **閾値0**: 積極走塁は不可能（実行不可）

走塁表の閾値は「走者の走力」×「外野手のT値（肩の強さ）」×「アウトカウント」×「守備形態」の組み合わせで決まる。

### 7.2.2 T値（外野手の肩の強さ）

| T値 | 意味 |
|-----|------|
| S | 最強の肩 |
| A | 強い肩 |
| B | 普通の肩 |
| C | 弱い肩（最弱） |

## 7.3 盗塁

### 7.3.1 概要

盗塁は**投球前**に完結する（打席とは独立したイベント）。

**流れ**:
```
攻撃側: 盗塁宣言（「盗塁」とチャットに記載）
守備側: 同意（「どうぞ」等）
攻撃側: スタート判定コマンド実行
  → 成功: エンド判定コマンド実行
  → 失敗: スタートできなかった判定。走者は動かず処理終了（以降の制約あり）
→ 結果確定後、通常の打席へ
```

### 7.3.2 スタート判定（st）

**コマンド**: `{背番号}-st{目標塁}`
例: `10-st2`（背番号10が2塁盗塁）

**判定ルール**: 補正後のst値 ≥ 1d20出目で成功

**st値補正**:

| 状況 | 補正 |
|------|------|
| 基本（二盗） | 補正なし |
| 左投手への二盗 | -2 |
| 三盗 | -4 |
| 本盗（ホームスチール） | -8 |
| クイック投法使用中の投手 | -2（三盗・本盗でも適用） |
| 出目「2」以下（二盗・三盗） | 補正を無視して成功 |
| 出目「1」（本盗） | 補正を無視して成功 |
| 出目「20」 | 牽制処理 |

**出目20（牽制処理）**:
- 二盗・三盗: 攻撃側がさらに1d20を振る。10以下→帰塁セーフ、11以上→牽制アウト
- 本盗: 自動牽制アウト（ダイス判定なし）

### 7.3.3 エンド判定（ed）

スタート成功後に実行する。

**コマンド**: `{背番号}-ed{目標塁}`
例: `10-ed2`（背番号10が2塁での成否判定）

**判定ルール**: ed値以下で成功（捕手T値加算後の1d20出目と比較）

**補正**:

| 状況 | 補正 |
|------|------|
| 基本 | なし |
| 捕手T値 | **-5〜+5の整数値**をed判定値にそのまま加算（外野手のT値S/A/B/Cとは全く別の概念） |
| クイック○の投手 | 捕手T値に+2を加えてから加算 |
| クイック✕の投手 | 捕手T値に-2を加えてから加算 |
| 本盗 | ed値に-8補正、捕手T値は**適用しない** |
| 出目「3」以下（二盗） | 無条件成功 |
| 出目「2」以下（三盗） | 無条件成功 |
| 出目「1」（本盗） | 無条件成功 |
| 出目「20」 | 無条件失敗 |

### 7.3.4 重盗（ダブルスチール）

1. **先頭走者のst判定**を実行
2. 成功後、**守備側が送球先を選択**（先頭走者か後続走者か）
3. 選択された走者のed判定を実行

> 守備側が送球先を選択する前にed判定を実行した場合は原則やり直し（両者合意で続行可）。

### 7.3.5 盗塁後の制約

| 状況 | 制約 |
|------|------|
| 盗塁成功後 | 次の塁へ進むまで再度盗塁指示不可 |
| スタート失敗後 | 打撃結果で進塁するまで再盗塁不可 |

### 7.3.6 盗塁後の作戦変更

| 状況 | 作戦変更 | バント |
|------|---------|--------|
| スタートを切った場合（成功・失敗問わず） | **両者可能** | 可能（**スリーバント補正-5**適用） |
| スタートを切れなかった場合 | **不可** | 可能（同じくスリーバント補正-5） |

## 7.4 バント

### 7.4.1 概要

バント宣言時は通常の打撃コマンドの代わりにバントマクロを使用する。

**バント判定値** = 打者のバント値（1〜10）+ 1d20出目 - 各種補正

### 7.4.2 バントマクロ体系

| マクロ | 使用条件 |
|--------|---------|
| `{背番号}-b-{塁}` | 通常バント（先頭走者がいる塁を指定） |
| `{背番号}-s-{塁}` | 内野前進守備時のバント（スクイズ含む） |
| `{背番号}-b3-{塁}` | スリーバント（通常守備） |
| `{背番号}-s3-{塁}` | スリーバント + 内野前進守備 |

> `{塁}` は先頭走者が位置する塁番号（1=1塁先頭、2=2塁先頭、3=スクイズ）。

### 7.4.3 補正

| 状況 | 補正 |
|------|------|
| スリーバント | -5（判定値に-5） |
| 内野前進守備時のスクイズ | -5（マクロが`-s-`に変わる＋-5） |

## 7.5 エンドラン

### 7.5.1 宣言条件

エンドランは以下の走者状況でのみ宣言できる:
- **走者1塁のみ**
- **走者1,3塁**

> 盗塁処理後（スタートを切った・切れなかった問わず）はエンドラン宣言不可。

### 7.5.2 処理フロー

1. 攻撃側が作戦発表タイミングで「エンドラン」を宣言
2. 守備側: 前進守備等を宣言（毎打席必須）
3. 投球マクロ → 打撃マクロ（通常の打席処理）
4. 打撃結果に応じてエンドラン変化処理（→7.5.3）を適用

### 7.5.3 エンドラン時の打撃結果変化

走者がスタートを切っているため、打撃結果が以下のように変化する。

**ゴロ系**

| 通常結果 | エンドラン時 | 理由 |
|----------|-------------|------|
| G#f | **G#aに変更** | スタート済みでフォースプレイ困難 |
| G#a | 変化なし | 元からアドバンス |
| G4D・G6D | 変化なし | 二遊間DPは成立 |
| その他のG#D（G1D/G3D/G5D等） | **G#aに変更** | 走者が動いているためDP困難 |
| 内野レンジでアウト | **LD（ライナー）扱い** | スタートした走者が戻れない |

**ヒット系**

| 通常結果 | エンドラン時 | 理由 |
|----------|-------------|------|
| H# | **H#aに変更** | スタート済みで余分に進塁 |
| H7a | **1塁走者が3塁に確定進塁** | スタート済み（通常は走塁表判定） |
| 2H#・2H#a | **H#aに変更** | 通常打撃とのトレードオフ |
| 3H# | **H#aに変更** | 同上 |
| HR# | **2H#aに変更** | 同上 |
| H8a | **G6Dに変更** | センター方向へのゴロ打球→遊撃がカバーして併殺 |

**特殊結果**

| 通常結果 | エンドラン時 | 処理 |
|----------|-------------|------|
| BB | **ファール扱いでやり直し** | エンドラン状態は解除される |
| DB | 変化なし | エンドラン無関係に通常出塁 |
| PO | 変化なし | 打者アウト・走者そのまま |

### 7.5.4 エンドランK（三振）時の処理

1. 打者は三振でアウト
2. 走者はスタートを切っているため**ed判定のみ**実施（st判定なし）
3. 判定基準: **走力×2**（個人のed値が低い場合はed値を採用）

> この処理が`pending_steal`フラグ（§5.9参照）の解決にあたる。

### 7.5.5 エンドラン状態の解除

以下の場合、エンドラン状態は解除され打席中の再宣言は不可:

- **UP表発生時**: 通常状態に戻してUP表処理。暴投・捕逸によるやり直しでも再宣言不可
- **BB発生時**: ファール扱いでやり直しになるが、エンドラン状態は解除

## 7.6 前進守備

### 7.6.1 種別と宣言

| 種別 | 宣言方法 | 発動条件 |
|------|---------|---------|
| 内野前進 | 「内野前進」 | **無死または1死で3塁走者が存在する**場合のみ宣言可 |
| 外野前進 | 「外野前進」 | 常に宣言可（走者なしでも可だが意味なし） |
| 内外野両前進 | 「内外野前進」 | 両者の条件を満たす場合 |

> **毎打席宣言必須**。宣言なし＝通常守備。サヨナラ場面でも省略不可。

**宣言タイミング**: 攻撃側の結果発表＋作戦発表の後。攻撃側は守備位置を見て作戦を変更できる。

### 7.6.2 内野前進守備の効果

内野前進守備時の打撃結果変化は §5（打席処理）で各コード別に記載。主なポイント:

- **G#a**: 3塁走者がいれば「ホームで刺す」選択が可能。打者はセーフ。ただしG1aは例外（通常と同じ）
- **G#f**: 1,3塁では打者アウト・3塁走者そのまま（スコアなし）
- **G#D**: G1Dを除き、打球が内野を抜けてヒット相当に（走者大量進塁）
- **バント（スクイズ）**: マクロが`-b-3`→`-s-3`に変更、判定値に-5補正

### 7.6.3 外野前進守備の効果

| 通常結果 | 外野前進時 | 理由 |
|----------|-----------|------|
| F#a（外野フライ犠牲） | **2H#aに変更** | 前進中のため頭上を越えて長打に |
| 2H#（二塁打） | **2H#aに変更** | 前進中のため打球処理が遅れ走者一掃 |

> 走塁表（積極走塁）判定において外野前進守備は「前進守備用走塁表（閾値が厳しい）」を参照する（→7.2.4）。

## 7.7 走塁保留の解決

打席処理（§5.9）で設定された保留フラグを解決する。

### pending_baserunning（走塁表保留）

1. 打席処理で走塁表対象フラグが立った
2. 攻撃側が「回す」か「止まる」を選択
3. 「回す」: `b-{外野手番号}` で閾値確認 → 1d20判定
4. 閾値以下でセーフ（進塁確定）、超過でアウト
5. 結果で走者状態を確定

### pending_steal（エンドランK時の盗塁保留）

1. エンドランK時に`pending_steal`フラグが立った
2. ed判定のみ実施（`{背番号}-ed{塁}`）
3. 判定基準: 走力×2（個人のed値が低い場合はed値を採用）
4. 成功: 走者進塁確定。失敗: 走者アウト

### at_bat_redo（暴投・捕逸による再打席）

1. UP表で暴投（WP）・捕逸（PB）が発生
2. 走者を1進塁させる
3. 同じ打者の打席を投球から再開
4. バント作戦だった場合はスリーバント補正（-5）が引き続き適用
