# 第5章: 打席処理

**状態**: 実装済み
**品質基準**: エミュレーターが本章の処理を実装できるレベル
**参照実装**: `thbig-irc-parser/src/thbig_irc_parser/game_state.py`, `atbat_parser.py`

---

## 5.1 概要

東方BIG野球の1打席は、以下の連鎖処理で構成される。

```
投手側: 投球コマンド（背番号-p）
        ↓
sian: 投球ナンバー出力（1〜5）
        ↓
打者側: 打撃コマンド（背番号-投球番号）
        ↓
sian: 打撃結果コード出力
        ↓
  ┌───────────────────────────────────┐
  │ 結果の分岐                         │
  │ (a) 通常結果コード → 5.4 直接処理  │
  │ (b) レンジ（CF等） → 5.5 レンジ    │
  │ (c) UP            → 5.6 UP表      │
  └───────────────────────────────────┘
        ↓
走者状況×結果コード → 走者・アウト・得点を更新
        ↓
走塁表判断（必要な場合） → 5.7
        ↓
次打席へ / 3アウトでチェンジ
```

---

## 5.2 投球番号の決定

守備側（投手保有チーム）が以下コマンドを実行する。

**コマンド**: `{背番号}-p`

sianボットが投球ナンバー（1〜5）を出力する。

### 特殊な投球ナンバー表記

| 表記 | 意味 | 使用する番号の決定方法 |
|------|------|----------------------|
| 通常 `3` | 投球番号3 | そのまま使用 |
| `3*` | 疲労状態の番号3 | 1段階劣化（番号-1）として扱う |
| `3/4` | 左右投手分岐 | 左投手→`3`、右投手→`4` |
| `3P1` | 走者有無分岐 | 走者なし→`3`、走者あり→`1` |

> **疲労ルール**: 疲労状態（`*`付き）の投球番号は1段階劣化する。元番号1は変化なし。
> 疲労の定義は §9（投球・疲労管理）を参照。

---

## 5.3 打撃マクロ実行

攻撃側（打者保有チーム）が以下コマンドを実行する。

**コマンド**: `{背番号}-{投球番号}`

例: 投球番号`3`が出た場合 → `39-3`（背番号39の選手で打撃番号3）

sianボットが**打撃結果コード**を出力する。

## 5.4 結果コード体系

### 5.4.1 アウト系

| コード | 意味 |
|--------|------|
| `K` | 三振 |
| `PO` | ポップアウト（内野フライ） |
| `LD` | ライナーアウト |
| `F{N}` | 外野フライ（浅い） |
| `F{N}a` | 外野フライ（犠牲フライ相当） |
| `G{N}f` | フォースゴロ |
| `G{N}a` | アドバンスゴロ（走者進塁型） |
| `G{N}D` | ダブルプレイゴロ |

### 5.4.2 安打系

| コード | 意味 |
|--------|------|
| `H{N}` | 単打（外野安打）。走塁表対象 |
| `H{N}a` | 単打（長め）。全走者確定2進塁 |
| `IH{N}` | 内野安打。積極走塁不可 |
| `2H{N}` | 二塁打。1塁走者が走塁表対象 |
| `2H{N}a` | 二塁打（長め）。全走者確定生還 |
| `3H{N}` | 三塁打。全走者生還 |
| `HR{N}` | 本塁打。全走者生還 |

### 5.4.3 出塁系

| コード | 意味 |
|--------|------|
| `BB` | 四球 |
| `DB`（`死球`） | 死球 |

### 5.4.4 分岐系（追加判定が必要）

| コード | 意味 |
|--------|------|
| `UP` | 好プレー珍プレー表（→5.6） |
| `CF`, `RF`, `LF`, `SS`, `P`, `1B` 等 | レンジチェック（→5.5） |

## 5.5 通常打席処理（走者状況×結果コード）

走者状況は8パターン: `empty` / `1` / `2` / `3` / `1,2` / `1,3` / `2,3` / `full`

### 5.5.1 K（三振）・PO（ポップアウト）・LD（ライナーアウト）

全走者状況共通: **打者アウト、走者変化なし**

> **LD例外**: レンジチェック結果のLDは「打者アウト＋先頭走者アウト」になる場合がある（→5.5節末尾）。

### 5.5.2 G#a（アドバンスゴロ）

**通常守備**

| 走者状況 | 処理 |
|---------|------|
| なし | 打者アウト |
| 1塁 | 打者アウト、走者→2塁 |
| 2塁 | 打者アウト、走者→3塁（**G5a・G6aは走者そのまま**） |
| 3塁 | 打者アウト、走者生還 |
| 1,2塁 | 打者アウト、各走者1進塁 |
| 1,3塁 | 打者アウト、各走者1進塁（3塁走者生還） |
| 2,3塁 | 打者アウト、各走者1進塁（3塁走者生還） |
| 満塁 | 打者アウト、各走者1進塁（3塁走者生還） |

**内野前進守備（G1aを除く）**

| 走者状況 | 処理 |
|---------|------|
| 3塁 | **3塁走者ホームでアウト**、打者1塁セーフ |
| 1,3塁 | **3塁走者ホームでアウト**、打者1塁セーフ、1塁走者→2塁 |
| 2,3塁 | **3塁走者ホームでアウト**、打者1塁セーフ、2塁走者→3塁 |
| 満塁 | **3塁走者ホームでアウト**、打者1塁セーフ、他走者1進塁 |
| その他 | 通常守備と同じ |

> **例外**: G1a（投手へのゴロ）は内野前進守備でも**常に通常守備と同じ処理**を適用。

### 5.5.3 G#f（フォースゴロ）

**通常守備**

| 走者状況 | 処理 |
|---------|------|
| なし | 打者アウト |
| 1塁 | 走者→2塁フォースアウト、打者1塁セーフ |
| 2塁 | 打者アウト、走者そのまま（**G3f・G4fは走者→3塁**） |
| 3塁 | 打者アウト、走者そのまま |
| 1,2塁 | 1塁走者→2塁フォースアウト、2塁走者→3塁、打者1塁セーフ |
| 1,3塁 | 1塁走者→2塁フォースアウト、3塁走者生還、打者1塁セーフ |
| 2,3塁 | 打者アウト、走者そのまま |
| 満塁 | 3塁走者→ホームフォースアウト、打者1塁セーフ、他走者1進塁 |

**内野前進守備**

| 走者状況 | 処理 |
|---------|------|
| 1,3塁 | **打者アウト**、1塁走者→2塁、**3塁走者そのまま**（通常と異なりスコアなし） |
| その他 | 通常守備と同じ |

### 5.5.4 G#D（ダブルプレイゴロ）

**通常守備**

| 走者状況 | 処理 |
|---------|------|
| なし | 打者アウト |
| 1塁 | 1塁走者+打者アウト（DP） |
| 2塁 | 打者アウト、走者そのまま |
| 3塁 | 打者アウト、走者そのまま |
| 1,2塁 | 1塁走者+打者アウト（DP）、2塁走者→3塁 |
| 1,3塁 | 1塁走者+打者アウト（DP）、3塁走者生還。G1D×無死: 通常DPまたは3塁走者挟殺を選択可。レンジGD: 3塁走者生還**せず** |
| 2,3塁 | 打者アウト、走者そのまま |
| 満塁 | 1塁走者+打者アウト、他走者1進塁。**G1D・レンジGD**: 3塁走者+打者アウト（ホームゲッツー）、他走者1進塁 |

**内野前進守備（G1D・レンジGDを除く）**

打球が内野手の間を抜け外野へ到達するため、打者はセーフ（ヒット相当）。

| 走者状況 | 処理 |
|---------|------|
| 2,3塁 | **H#a処理**（全走者2進塁＝全員生還） |
| 満塁 | **3塁走者ホームフォースアウト**、他走者1進塁、打者1塁セーフ |
| その他 | **H#処理**（全走者1進塁、打者1塁セーフ） |

> **例外**: G1D（投手ゴロDP）は内野前進守備でも通常守備と同じ処理。  
> **レンジGD**: 内野前進守備でも「抜かれない」扱い（通常守備参照）。

### 5.5.5 F#（外野フライ）・PO（内野フライ）

全走者状況共通: **打者アウト、走者変化なし**

### 5.5.6 F#a（犠牲フライ相当）

| 走者状況 | 処理 |
|---------|------|
| なし / 1塁 / 2塁 / 1,2塁 | 打者アウト、走者変化なし |
| 3塁 / 1,3塁 / 2,3塁 / 満塁 | 打者アウト、**3塁走者生還** |

> **注**: F#a（F7a/F8a/F9a）では、2塁走者のタッチアップ進塁（3塁→本塁）も走塁表で判定可能（F8a/F9aの特殊パターン）。詳細は §7（走塁）参照。

### 5.5.7 H#（単打・外野安打）

| 走者状況 | 処理 |
|---------|------|
| 全走者状況共通 | ヒット、全走者1進塁 |

> **積極走塁**: 攻撃側が「回す」を選択した場合、走塁表判定（→5.7）を実施。  
> **例外**: H7（左翼方向）の1塁走者は走塁表対象**外**（1進塁で止まる）。

### 5.5.8 H#a（単打・長め外野安打）

| 走者状況 | 処理 |
|---------|------|
| なし | ヒット |
| 1〜満塁 | ヒット、全走者**確定2進塁** |

> **H7a例外**: 1塁走者のみ確定1進塁＋走塁表対象（2進塁は保証されない）。2塁・3塁走者は通常のH#aと同じ（確定2進塁）。

### 5.5.9 2H#（二塁打）

| 走者状況 | 処理 |
|---------|------|
| なし | 2塁打 |
| 1〜満塁 | 2塁打、全走者**確定2進塁** |

> **1塁走者の積極走塁**: 攻撃側が「回す」を選択した場合、1塁走者のみ走塁表判定（→5.7）。

### 5.5.10 2H#a（長い二塁打）・3H#（三塁打）・HR#（本塁打）

| 走者状況 | 処理 |
|---------|------|
| 全走者状況共通 | 2塁打/3塁打/本塁打、**全走者生還** |

### 5.5.11 BB（四球）・DB（死球）

| 走者状況 | 処理 |
|---------|------|
| なし〜3塁 | 打者→1塁。フォース状態でない走者はそのまま |
| 1塁（フォース） | 打者→1塁、1塁走者→2塁 |
| 1,2塁 | 打者→1塁、1塁走者→2塁、2塁走者→3塁 |
| 満塁 | 打者→1塁、全走者1進塁（3塁走者生還） |

> フォース状態の走者（塁が埋まっている場合）のみ強制進塁。フォースでない走者は進塁しない。

### 5.5.12 IH#（内野安打）

| 走者状況 | 処理 |
|---------|------|
| 全走者状況共通 | 内野安打、全走者1進塁 |

> **積極走塁不可**: IH#は走塁表の対象外。走者は必ず1進塁のみ。

## 5.6 レンジチェック処理

打撃結果コードが守備位置名（`CF`, `RF`, `LF`, `SS`, `P`, `1B` 等）だった場合に発生する。

**手順**:
1. 結果の守備位置から対象野手を特定
2. 対象野手の**守備力（数値）**と**エラーランク（S/A/B/C/D/E）**を参照
3. `内野レンジ{N}` または `外野レンジ{N}` マクロを実行
4. sianボットが新たな打撃結果コードを出力
5. → 5.4の通常処理に合流

**内野レンジ主要出力**:

| result | 意味 |
|--------|------|
| `H` / `2H` | レンジオーバー → ヒット/二塁打扱い（対応外野手が処理: 1B→RF, 3B→LF, P/2B/SS→CF） |
| `IH` | 内野安打 |
| `GD` | ダブルプレイ（レンジGD） |

> **投手レンジ補足**: 結果が`2H`の場合は`H`として扱う（二塁打にはならない）。

**外野レンジ主要出力**:

| result | 意味 |
|--------|------|
| `H` | レンジオーバー → ヒット扱い |
| `Fa` | 外野フライ（通常アウト） |

**LD（ライナー）発生時**:
- 打者アウト
- 先頭走者も**アウト**（例外: 帰塁できた場合は除く）

> **レンジチェックからUP表へは遷移しない。** UP表はUP結果コードからのみ発生する。

## 5.7 UP表処理

打撃結果コードが `UP` の場合、好プレー珍プレー表を参照する。

**コマンド**: `打:up`

**4桁コード生成**: 1つ目d20（千・百の位） × 2つ目d20（十・一の位）

例: [19][07] → コード `1907`

### 5.7.1 主要パターン

| コード範囲 | 処理 |
|-----------|------|
| 0101〜0920 | エラーチェック（→5.7.2） |
| 1001〜1002 | 雨天（ノーゲーム判定） |
| 1003〜1004 | 雨天（コールド判定） |
| 1005〜1010 | 死球 → DBと同じ処理 |
| 1011〜 | 特徴・暴投・捕逸・その他（→5.7.3） |

**球場設定による分岐**:
- コード 1911〜1912 → 球場設定 **1112** 適用
- コード 1913〜1914 → 球場設定 **1314** 適用

### 5.7.2 エラーチェック

UP表からのみ発生（レンジチェックからは発生しない）。

| コード範囲 | 対象野手 |
|-----------|---------|
| 0101〜0204 | 投手（P） |
| 0205〜0220 | 捕手（C） |
| 0301〜0320 | 一塁手（1B） |
| 0401〜0504 | 二塁手（2B） |
| 0505〜0620 | 三塁手（3B） |
| 0701〜0816 | 遊撃手（SS） |
| 0817〜0904 | 左翼手（LF） |
| 0905〜0912 | 中堅手（CF） |
| 0913〜0920 | 右翼手（RF） |

**エラーランクSの場合**: チェック不要、確定アウト

**内野エラー判定結果（3パターン）**:

| 結果 | 処理 |
|------|------|
| 処理成功 | G#fと同じ処理（フォースゴロ） |
| ファンブル | 打者・走者1進塁 |
| 悪送球 | 打者・走者2進塁 |

**外野エラー判定結果（3パターン）**:

| 結果 | 処理 |
|------|------|
| 処理成功 | F#と同じ処理（アウト） |
| 落球 | 打者・走者1進塁（2死時: 走者のみ2進塁） |
| ワンヒットワンエラー | 打者・走者2進塁（打者に安打記録） |

### 5.7.3 特徴・暴投・捕逸

**結果置換型**（打席結果をその結果に差し替える）:

| UP結果 | 処理 |
|--------|------|
| 投手ホームラン | HR処理 |
| 野手三振 | K処理 |
| 死球（特徴） | DB処理（この死球では怪我・危険球退場なし） |
| 内野安打（特徴） | 打者1塁セーフ。1塁走者がいれば2塁フォースアウト。内野前進守備時は3塁走者ホームアウト |

**走者進塁型**（走者を進め、投球からやり直し）:

| UP結果 | 処理 |
|--------|------|
| 暴投（WP） | 全走者1進塁、投球やり直し。走者なしなら投球やり直しのみ |
| 捕逸（PB） | 全走者1進塁、投球やり直し。走者なしなら投球やり直しのみ |
| ボーク | 全走者1進塁、投球やり直し。走者なしなら投球やり直しのみ |

> やり直し後の再投球でバント作戦だった場合、スリーバント補正（-5）が適用される。  
> 「WPしない」投手特殊能力: 暴投時は走者進塁なしで投球やり直しのみ（捕逸は防げない）。

**振り直し型**（特定の列で打撃をやり直す）:

| 特徴 | 発動側 | 処理 |
|------|--------|------|
| 対右 | 投手 | 打者が右打者なら列5、左打者なら列1で振り直し |
| 対左 | 投手 | 打者が右打者なら列1、左打者なら列5で振り直し |
| 対右 | 野手 | 相手投手が右なら列1、左なら列5で振り直し |
| 対左 | 野手 | 相手投手が右なら列5、左なら列1で振り直し |
| 1C5 | 野手 | 走者なし→列1、走者あり→列5で振り直し |
| チャンス | 野手 | 走者なし→列5、走者あり→列1で振り直し |

### 5.7.4 暴投追加処理（4-19ルール）

通常の暴投頻度バランス調整のため追加されたルール。

**発生条件（全て満たす場合）**:
- 無死または1死
- 走者が存在する
- 打撃結果がK（三振）
- 投球番号4・出目19のKであること

**処理**: 暴投発生（走者1進塁・投手に暴投記録） → その後打者は三振

**「暴投追加」特殊能力を持つ投手**: 投球番号5・出目19でも発生（5-19）。
ただし打者の投球番号5が全K（マクロを振らない処理）の場合は発生しない。

> 「WPしない」投手は4-19でも暴投なし（普通に三振）。

## 5.8 走塁表判断（積極走塁）

打撃処理後、以下の条件で積極走塁（より先の塁を狙う）判断が発生する。

### 5.8.1 走塁表対象となる条件

| 結果コード | 走塁表の対象 |
|-----------|------------|
| H#（外野安打） | 全走者（**ただしH7の1塁走者は対象外**） |
| H#a | H7a: 1塁走者のみ対象 |
| 2H#（二塁打） | 1塁走者のみ対象 |
| F#, F#a（外野フライ） | 3塁走者（F#aなら確定生還） |
| F8a/F9a | 2塁走者も走塁表対象（3塁→本塁判定） |
| IH# | **対象外**（積極走塁不可） |

### 5.8.2 走塁表判定手順

1. 攻撃側が「回す」または「止まる」を選択
2. 「回す」の場合: 走塁表マクロ（`b-{外野手番号}`）で閾値を確認 → 1d20を振る
3. **d20出目 ≤ 閾値 → セーフ（進塁成功）**
4. **d20出目 > 閾値 → アウト**
5. 閾値が`0` → 積極走塁は不可能（実行不可）

> **複数走者の場合**: 先頭走者のみ判定（IRC実運用の慣例）。  
> **2死時**: ライナーバック不要のため閾値が緩和される（2死用の走塁表を参照）。

走塁表の詳細（閾値・守備形態別補正等）は §7（走塁）参照。

---

## 5.9 保留フラグと後続処理

一部の打席は1打席で完結せず「保留フラグ」を立てて後続処理に委ねる。

| フラグ | 発生条件 | 後続処理 |
|--------|---------|---------|
| `pending_baserunning` | 走塁表判断が必要なH#等 | 走塁表で判定後に状態確定 |
| `at_bat_redo` | WP/PBで同打者が再打席 | 投球から同打者の打席を再開 |
| `pending_steal` | エンドランK時に1塁走者の判定が必要 | ed判定（→§7.3）で解決 |

---

## 5.10 アウトカウントとイニング終了

- 3アウトで攻守交替（チェンジ）
- 最終回（9回裏等）に後攻がリードした時点で試合終了
- イニング終了時にアウト・スコア・走者状況のリセット
