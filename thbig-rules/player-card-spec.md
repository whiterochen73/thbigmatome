## 1. カード全体レイアウト

### カード構成

選手カードは以下の要素で構成される固定フォーマット:

| 要素 | 説明 | 値域 | 備考 |
|------|------|------|------|
| チーム名 | カード所属チーム | TURTLES/FROGS/ARROWS/UNICONES/FOX/QUEENBEES + ハチナイコラボ等 | 出典作品別 |
| 背番号 | 選手の背番号 | 数値（東方）/ P○・F○・T○・A○・O○（コラボ）/ 3桁数値（PM） | 詳細は§4参照 |
| ポジション | 守備位置 | P(投手)/C(捕手)/1B/2B/3B/SS/OF(外野手) | 複数ポジション可 |
| 利き腕 | 投球・打撃の方向 | 右/左 | 投手と野手で独立 |
| **打撃データ** | 5列×20行の打撃結果表 | 野手:100マス / 投手(通常):80マス / 投手(5\|P):100マス | 投球ナンバー(1-5)×出目(1-20)。詳細は下記参照 |
| **投球データ（P列）** | 投手のみ: 出目1-20に対応の投球ナンバー1-5 | 20個 | 野手には存在しない |
| 盗塁力 | スタート値 + 成功値 | 数値 | 二盗・三盗・ホームスチール別 |
| バント力 | バント能力 | 1-10 | 低=苦手, 10=名手 |
| 走力 | 走塁能力 | 1-5（通常）/ 6（特殊） | 走力6は「風神少女」能力発動時のみ |
| **守備力** | ポジション別の守備スペック | 複数値 | 詳細は§4参照 |
| 疲労P（スタミナ） | 投手の投球可能回数 | 0-9 / 先発/リリーフ別 | 詳細は§5参照 |
| 特徴 | キャラ固有の好プレイ珍プレイ効果 | 複数可 | 実装上は「特徴」と「特殊能力」を区分 |
| けがレベル | 回復力（高いほど軽傷） | 1-7 | 1=重傷, 7=フルイニング |

### 打撃マス数詳細

- 野手: 5列×20行 = 100マス
- 投手(通常): 4列×20行 = 80マス（投球5の打撃列は省略）
- 投手(5|P併記): 5列×20行 = 100マス
  - 5|P = 投球番号5の打撃結果が存在する投手のマーキング
  - 単独Kのマスは結果表記が省略され投球番号のみ表示 → パーサーは「番号のみ=K」と解釈

### 利き腕

- 投球（右投/左投）と打撃（右打/左打/両打）は独立
- 「両投」は現時点で未存在
- パーサーフィールド: `throwing: 右/左`, `batting: 右/左/両`

### 特徴欄ラベル体系

- 【投手】: 投手用特徴
- 【野手】: 野手用特徴
- 【特例起用】: 特例選手指定時に参照する特徴
- 特殊能力は特徴より上に記載される傾向（現時点で明確な区別は不要）

### 二刀流選手の判別

- 特徴欄に【投手】と【野手】の両ラベルが存在 = 二刀流

### フォーマット

- **配布形式**: Excel → PDF変換（元Excelは入手不可）
- **1ページ構成**: 固定9人（背番号順またはポジション順）
- **読み取り方法**: PDF裁断 → 高解像度画像化 → 文字・数字読み取り
  - 参照スクリプト: `split_card_pdf.sh` + `trim_card_images.sh`

### カード層別構造（実装上の分類）

```
┌─ Card整体構造 (CardSet)
│  ├─ Player（選手情報）
│  │  ├─ 基本情報（背番号, ポジション, チーム, 利き腕等）
│  │  └─ パラメータ（守備値, 走力, バント値等）
│  │
│  └─ PlayerCard（データシート）
│     ├─ 打撃データ（BattingCardData）
│     │  ├─ BattingResult[] (5×20=100マス)
│     │  └─ 出目別優先順（メタデータ）
│     │
│     ├─ 投球データ（PitchingCardData）- 投手のみ
│     │  ├─ PitchingNumber[] (20個)
│     │  └─ 疲労P別スラッシュ(例: 7/2)
│     │
│     ├─ 特徴（FeaturesData）
│     │  └─ Feature[] (0-複数)
│     │
│     └─ けがレベル（InjuryLevel）
```

## 2. 投球カード（P列）

### 概要

投手が持つデータシート。出目1-20に対応した投球ナンバー1-5の並び。
投手の能力を決定する最重要パラメータ。

### 投球ナンバーの意味

| 番号 | 意味 | 打者側影響 | 投手評価 |
|------|------|----------|----------|
| **1** | 絶好球 / 甘い球 | 長打されやすい | 悪い |
| **2** | そこそこ打てる球 | 打てるが難度あり | 悪い |
| **3** | 打たせてとる球 | アウト取りやすい | 良い |
| **4** | ボール球 | 四球可能性 | 悪い |
| **5** | 三振をとる球 | 最強（ほぼK） | 良い |

### P列の構造

#### 基本配置
- 出目が小さいほど（1に近いほど）、投球番号も小さい傾向
- 末尾側（出目18-20）に高い投球番号が集中しやすい
- `/` (スラッシュ) や `P` (左右分岐) は末尾側に配置

#### 疲労マーカー「*」
- **記号**: `*` を投球番号に付加（例: 2*）
- **効果**: 疲労状態の投手が該当投球を使用時、1段階劣化
  - `2*` → `1` に劣化
  - `3*` → `2` に劣化
  - `5*` → `4` に劣化
- **出現制限**: 単独の投球番号2・3・5にのみ付加（例: 全て2*, または3と5に1個ずつ等）
- **上限**: 合計1-4個（疲労耐性により異なる）

#### スラッシュ「/」形式（投手の対左/対右分岐）
- **記号**: `投球1/投球2` (例: 1/3)
- **意味**:
  - `/` 左側 = 対左打者（打者が左打者のとき）
  - `/` 右側 = 対右打者（打者が右打者のとき）
- **同腕対決**: 投手有利になるよう設定

#### P分岐（走者有無条件）
- **記号**: `投球P投球` (例: 1P3)
- **意味**:
  - `P` 左側 = 走者なし時の投球
  - `P` 右側 = 走者あり時の投球

### P列例外選手

#### 稀神サグメ（きじんサグメ）
- **特徴**: 投球番号が上下逆（5→1の順）
- **読み方**: 通常と逆の優先順位でデータ抽出・バリデーション

#### 東風谷早苗（こちや さなえ）
- **特徴**: 能力が高く、疲労マーカー「*」が6個（通常上限4超え）
- **読み方**: 「*」の上限チェックで例外扱い

### バリデーション（P列用）

**暗黙ルール**:
1. 出目が小さいほど小さい投球番号が出やすい（傾向）
2. `/` や `P` は末尾側に配置される傾向
3. `*` は単独の2・3・5にしかつかない（同番号内で上に配置）
4. 「*」の合計数 = 1-4個以内

**推測・修復手順**:
- 読み取り失敗マスがある場合、上記規則から推測可能
- 完全に欠落した投球番号がないか確認

## 3. 打撃カード

### 概要

野手が持つ100マス（5投球ナンバー × 20出目）の打撃結果表。
投手も持つが、投球ナンバー5は全て K（三振） で、他の4列のみ有意。
（投手の打撃 = 野手能力が低い設定）

### 出現優先順（極めて重要）

**優先順序**（高い順位から埋まる傾向、同一マスは同優先順位）:

```
1. HR# (本塁打)
2. 3H# (三塁打)
3. 2H# (二塁打)
4. H# (単打)
5. IH# (内野安打)
6. レンジ (守備判定マス)
7. G# (ゴロ)
8. F# (フライ)
9. PO (ポップフライ)
10. BB (四球)
11. DB (死球)
12. K (三振)
```

### 打撃データの並び順（投球ナンバー別）

#### 投球1（甘い球） - 長打が多い
- HR1, HR2, ... (ホームランが集中)
- 例: 長打力評価 = 投球2のHR数（詳細は§6参照）

#### 投球2（そこそこ打てる球）
- HR, 3H, 2H, H が混在
- **長打力評価の基準**（P情報）

#### 投球3（打たせてとる球）
- H（単打）が多い
- **巧打力評価の基準**（投球3のH有無+対応度）

#### 投球4（ボール球）
- BB（四球）が主
- DB（死球）も原則この列にのみ

#### 投球5（三振球）
- 通常は全て K（三振）
- **三振耐性評価の基準**

### 出目別レンジ配置

**固定分布パターン**:

| 投球番号 | レンジ出目 | レンジ個数 | 例 |
|---------|-----------|-----------|-----|
| 1 | (複数) | 2個 | 出目7-8など |
| 2 | (複数) | 3個 | 出目10-12など |
| 3 | (複数) | 3個 | 出目15-17など |
| 4 | (複数) | (ボール球優先) | — |
| 5 | (複数) | (三振優先) | — |

### K以降全Kルール

**重要なバリデーションルール**:

- K（三振）が単独出現した最初の出目以降、19まで全てK
- 例: 投球3で出目12がKなら、13-19も全てK
- **例外**: 出目20は UP（好プレイ珍プレイ表）参照
- 上述のルール適用外: 投球5が全Kの選手は出目20もK（UP表参照しない）

### 出目20 = UPルール

通常は **出目20 = UP表参照** で特殊イベント判定。

**例外**:
- 投球ナンバー5が全マス三振（確定三振選手）の場合 → 出目20もK

### 打撃カード例外選手

#### 鬼人正邪（きじん しょうじゃ）
- **特徴**: 打撃結果の順番が逆転（出目1にUPが来る等）
- **能力再現**: キャラの「ひっくり返す」能力の反映
- **読み方**: 出目20→1の優先順に並び直すイメージで読み取り

### バリデーション（打撃カード用）

**暗黙ルール**:

1. **優先順の厳格性**: 高優先度の結果から埋まる（100マスを優先度順に割り当て）
2. **K以降全K**: 単独Kが出た出目以降19までは全てK
3. **出目20の特殊性**: 通常UP、投球5全Kの場合のみ出目20もK
4. **レンジマスの位置**: 投球別に固定分布（投球1に2個、2と3に3個ずつ）
5. **スラッシュ位置**: 左右分岐は記載ある場合のみ、複数箇所での判別が必要（スイッチヒッター判定）

### スイッチヒッターの打撃カード

- **定義**: 左右両打者の打撃データを持つ選手
- **読み方**: 1箇所だけでは判別不可、複数箇所のスラッシュから推測
- **実装処理**: 投手のスラッシュ投球を受けた場合、打者に有利な方を自動適用

## 4. 守備・走力・その他パラメータ

### 守備力スペック

各ポジションごとに記載される守備評価。

| 要素 | 説明 | 値域 | 用途 |
|------|------|------|------|
| **ポジション番号** | 守備位置の区分 | P(1)/C(2)/1B(3)/2B(4)/3B(5)/SS(6)/LF(7)/CF(8)/RF(9) | レンジチェック等で使用 |
| **レンジ値** | 守備範囲の広さ | 0-5（0=狭い, 5=広い） | レンジチェック判定の基準値 |
| **エラーランク** | 守備の確実性 | S/A/B/C/D/E（S=最高, E=最低） | エラーチェック時の DP/ファンブル/悪送球判定 |
| **T値（送球能力）** | 肩の強さ | S/A/B/C (文字) or ±数値 (数値) | 外野・捕手のみ。走塁判定・盗塁判定で加算補正 |

#### ランクS（エラーランク特殊）

- **定義**: ルールブック未記載の特殊ランク
- **付与対象**: シーズン通じ多数試合出場かつエラー無しの選手
- **効果**: エラーチェック不要で確定アウト（e判定省略）
- **実装注意**: エラーランクの値域に S を含める必要あり

#### T値の記載

- **外野手**: 肩の強さ（文字S-C, 数値±表現）
- **捕手**: 盗塁阻止能力（T+数値形式）
- **その他ポジション**: 記載なし（実装上は 0 扱い）

#### 守備値カード記載フォーマット

カード上での実際の表示形式:
- 1行目: `{ポジション}: {レンジ値}{エラーランク}` (例: `C: 3A`, `OF: 4C`, `2B: 4A`)
- 2行目（捕手・外野手のみ）: `T: {値}`
  - 捕手: 数値±表記 (例: `T: +1`)
  - 外野手: ランク (例: `T: B`)
- 内野手: T行なし

#### 外野守備値の個別表記

- OF（統合）vs LF/CF/RF/LR/LC/CR（個別）は排他的
- 作品で分けるのではなく、カード記載で判別
- LR=両翼, LC=レフト+センター, CR=センター+ライト
- T値はそれぞれのポジション直下に記載

#### 複数ポジション

- 守備値が複数ポジション分記載される場合がある
- ポジションの記載順は固定ではない
- T値は該当ポジションの直下に配置される

### 走力

| 値 | 説明 | 走塁・盗塁への影響 |
|----|------|------------------|
| 1 | 遅い | 走塁表の成功値が厳しい |
| 2-4 | 通常 | 平均的な判定 |
| 5 | 速い | 走塁表の成功値が有利 |
| 6 | 特殊 | 「風神少女」能力発動時のみ。走塁表-2補正 |

**特殊能力「風神少女」**（射命丸文）:
- 発動時のみ走力 6 に到達
- 走塁表のダイス目に-2補正（ただし出目20は補正なし、自動アウト）

### バント値

| 値 | 説明 | バント成功率 |
|----|------|-----------|
| 1-3 | 苦手 | 低 |
| 4-7 | 通常 | 中程度 |
| 8-10 | 名手 | 高（ただし前進守備やスリーバント補正で-5されると厳しい） |

**計算式**:
```
バント判定値 = 打者のバント値 + 1d20出目
              - スリーバント補正（-5）
              - 前進守備補正（-5）
```

### けがレベル（怪我レベル）

| 値 | 説明 | 回復速度 | 出場制限 |
|----|------|--------|--------|
| 1 | 重傷 | 遅い（多日数） | ほぼ不可 |
| 2-5 | 軽傷～中傷 | 可変 | 日数に応じた制限 |
| 6 | 全試合出場 | 早い | ほぼなし |
| 7 | フルイニング | 最速 | なし（連続出場可） |

**カード上の読み取り方**:
- 値1〜6: 「特徴 3」のように数字で記載
- 値7(フルイニング): 数字ではなく「フルイニング」と文字で記載
- パーサーは数字と「フルイニング」文字列の両方を扱う必要あり

**豆知識（P情報）**:
- 元々はプロ野球版で「シーズン出場試合数」に基づく設定
  - 6 = 全試合出場
  - 5 = 数試合欠場
  - 4 = レギュラークラス
  - 3 = 100試合前後
- 東方版では各キャラの設定で割り振り
- 実際のプロ野球シーズン記録に基づいている

### 特徴けが日数ルール（＊ルール）

特徴欄に記載されるけが免除・日数指定のルール:

- ＊ = 免除回数（怪我せず判定終了）、上から順に消化
- ＊なしの日数 → その日数分負傷
- （するごと）付き日数 → 毎回その日数の怪我（消化されず繰り返し）
- 日数なしの（するごと）→ けがレベルで怪我チェック（特徴けが扱い、試合前負傷軽減ルール適用外）
- 全エントリ消化後 → カード記載のけがレベルでの通常判定に戻る

### コラボ選手と背番号

| 種別 | 背番号 | 接頭辞 | 備考 |
|------|--------|--------|------|
| 東方選手 | 数値 | なし | 通常の背番号 |
| ハチナイ投手版 | P+数値 | P | 同一選手の投手/野手版が存在しうる |
| ハチナイ野手版 | F+数値 | F | 同上 |
| 球詠(たまよみ) | T+数値 | T | 川崎稜は1年時/2年時の2種存在 |
| エイプリル | A+数値 | A | カードに「(エイプリル)」表記。例外: リリーブラック(186) |
| 完走カード | O+数値 | O | 1シーズン完走チームから1人。チーム名=所属チーム。実成績ベース |
| PMカード | 3桁数値(0始まりあり) | なし | チーム名=GLADIATS。東方と番号重複なし |

※横綱カード、モブ設定カードは公式戦スコープ外

**実装注意**:
- 背番号フィールドは **文字列型** で保持する必要がある

## 5. 疲労P（スタミナ）

### 概要

投手の投球可能回数。疲労システムの中核。

### 疲労P値の範囲と判別

| 値 | 判別 | 意味 |
|----|------|------|
| 0-3 | リリーフ | 中継ぎ投手（短期間登板） |
| 4-9 | 先発 | 先発投手（長期間登板） |

**判別ルール**:
- 数字1つのみの場合: 値で判別（0-3=リリーフ, 4-9=先発）
- 複数パターンの場合: スラッシュで記載

### スラッシュ形式（先発/リリーフ区分）

**記号**: `先発値/リリーフ値`
- **例**: `7/2`（先発時7, リリーフ時2）
- **意味**: 先発と中継ぎで異なるスタミナを持つ投手

### R付き制約（リリーフ専任）

**記号**: `疲労値R` (例: 1R)
- **効果**: リリーフ専任。他に先発可能投手がいる場合、先発不可
- **例外**: オープナー ルール（上級選択ルール）でのみ先発可能

### C付き制約（クローザー）

**記号**: `疲労値C` (例: 3C)
- **効果**: 9回以降のみ登板可能
- **判別**: セーブ機会の有無は関係なし、純粋に9回以降制限
- **例**: 紫、隠岐奈

## 6. 選手評価の読み方

### 長打力（ホームランパワー）

**基準**: 投球ナンバー2のHR数

| HR個数 | 評価 | 本塁打力換算 |
|--------|------|-----------|
| 0個 | なし | 低い |
| 1個 | 標準 | 24本級（通常） |
| 2個 | 高い | 38本級 |
| 3個+ | 極高 | 55本級以上 |

### 巧打力（コンタクト能力）

**基準**: 投球ナンバー3の対応度

| 対応 | 評価 | 打率換算 |
|-----|------|--------|
| 3-1確定ヒット + 3-2スラッシュあり | 超高 | .330以上 + タイトル級 |
| 3-1レンジあり | 高い | .250-.300 |
| 3-1がK or 結果なし | 低い | .200以下 |

### 三振耐性（三振回避能力）

**基準**: 投球ナンバー5の構成

| 構成 | 評価 | 三振率換算 |
|-----|------|----------|
| 全てK（確定三振） | 低い | 30%+ |
| K多数 + 長打あり | 中 | 20-30% |
| Kと他結果混在 | 高い | 15-20% |

**例外**: 投球5に打撃結果が用意されている投手や、5は全Kでも他番号で打力が高い投手も存在

### 走塁能力と特徴

- **走力**: 前述の§4参照
- **盗塁力**: st値+ed値から判定（高い=スタート良い+成功率高い）
- **バント力**: 前述の§4参照

## 7. バリデーションルール（実装参考）

### チェック項目一覧

#### レベル1: 必須チェック

- [ ] 背番号: 有効範囲（1-181、またはコラボ接頭辞）
- [ ] ポジション: P/C/1B/2B/3B/SS/OF のいずれか（複数可）
- [ ] 打撃データ: 5列×20行 = 100マス埋まっているか（投手は80マス）
- [ ] 投球データ: 投手なら20個のP列値が存在するか
- [ ] 疲労P: 0-9 の範囲、またはスラッシュ形式 `N/N`

#### レベル2: ルール整合性チェック

- [ ] 優先順: 高優先度の結果（HR→3H→...→K）が上位出目から順序よく配置か
- [ ] K単独→以降全K: Kが初出現した出目より後、出目19までが全てKか
- [ ] 出目20ルール: 通常はUP、投球5全Kの場合はK
- [ ] 疲労*: 単独2・3・5にのみ付加、合計1-4個か
- [ ] レンジ分布: 投球1に2個、2と3に3個ずつか（ただし例外あり）

#### レベル3: 特殊記号チェック

- [ ] スラッシュ「/」: 左右分岐の適切性、末尾側配置か
- [ ] P分岐「P」: 走者有無の条件分岐が適切か
- [ ] 例外選手フラグ: サグメ（逆順）、正邪（逆転）、早苗（*6個）等の特定

#### レベル4: 参照表の一貫性

- [ ] 打撃結果コード: 記載の記号が打撃結果表に存在するか（K/BB/DB/HR/3H/2H/H/IH/G#/F#/PO等）
- [ ] レンジマス: 記載ポジション番号が妥当か（P/1B/2B/3B/SS/LF/CF/RF）
- [ ] UP表記号: UP コードが好プレイ珍プレイ表に存在するか

def validate_batting_card(card_data):
    """打撃カード仕様チェック"""
    errors = []

    # レベル1: 基本構造
    if len(card_data['batting']) != 100:
        errors.append("打撃マス数が100ではない")

    # レベル2: K以降全K
    for pitch_num in range(1, 6):
        column = [card_data['batting'][20*(pitch_num-1)+i] for i in range(20)]
        k_first = next((i for i, cell in enumerate(column) if cell == 'K'), None)
        if k_first:
            if not all(cell == 'K' for cell in column[k_first+1:19]):
                errors.append(f"投球{pitch_num}: K以降全Kルール違反")

    # レベル3: 優先順の妥当性
    priority_order = ['HR', '3H', '2H', 'H', 'IH', 'レンジ', 'G', 'F', 'PO', 'BB', 'DB', 'K']
    # ... (優先度チェック省略)

    return errors if errors else "OK"
