# レンジチェック表
出典: https://thbigbaseball.wiki.fc2.com/wiki/%E3%83%AC%E3%83%B3%E3%82%B8%E3%83%81%E3%82%A7%E3%83%83%E3%82%AF%E8%A1%A8
取得日: 2026-02-20

## 概要
守備範囲判定システム。打撃結果が「レンジチェック」（P/1B/2B/3B/SS/LF/CF/RF）となった場合に使用。
野手のレンジ値（守備範囲 0〜5）に基づき、d20を振って打球種別（プレー結果）を決定する。

## 使い方
1. 打撃結果が守備位置（例: SS）を示す → その野手でレンジチェック
2. 野手のレンジ値（0〜5）を確認
3. d20を振る
4. 下表でレンジ×ダイス目 → **プレー種別** を確認
5. プレー種別を打撃結果表で処理（エラーチェックはUP表処理からのみ発生。レンジチェック結果から直接エラーチェックへは遷移しない）

### プレー種別の意味（内野）
| 種別 | 意味 | 処理 |
|------|------|------|
| 2H  | 二塁打方向に抜ける | 二塁打として処理 |
| H   | 単打方向に抜ける | 単打として処理 |
| IH  | 内野安打 | 内野安打として処理 |
| Ga  | アドバンスゴロ | バッターアウト、走者進塁可 |
| Gf  | フォースゴロ | フォースプレイ |
| GD  | ダブルプレイゴロ | 併殺機会 |
| LD  | ライナー | ライナーアウト |

### プレー種別の意味（外野）
| 種別 | 意味 | 処理 |
|------|------|------|
| 3H  | 三塁打方向に抜ける | 三塁打として処理 |
| 2H  | 二塁打方向に抜ける | 二塁打として処理 |
| H   | 単打方向に抜ける | 単打として処理 |
| Fa  | 犠牲フライ | バッターアウト、3塁走者生還可 |
| F   | 通常フライ | バッターアウト |

**「-」** = このレンジではそのプレー種別に到達不可（ダイス目の範囲外）

### 位置別の処理ルール（内野手）
- **投手・二塁手・遊撃手**：抜けた打球はセンター方向（中堅手レンジで追加判定）
- **一塁手**：抜けた打球はライト方向（右翼手レンジで追加判定）
- **三塁手**：抜けた打球はレフト方向（左翼手レンジで追加判定）

## 注意事項
- バント時はこのレンジチェック表を使わない（bunt_table.md参照）
- 各ランク行のダイス範囲の合計は必ず20になる（全プレー種別を網羅）
- レンジ0では Ga/Gf/GD/LD は不可能（内野）、Fa/F は不可能（外野）

## IRC実装メモ
- 打球結果で守備位置（例: SS）を取得
- 野手のレンジ値（0〜5）を参照
- d20を振り、表を参照してプレー種別を決定
- 次ステップ: プレー種別を打撃結果表で処理
- アウト系（Ga/Gf/GD/LD/Fa/F）はそのままアウト処理（エラーチェックはUP表処理からのみ発生）
