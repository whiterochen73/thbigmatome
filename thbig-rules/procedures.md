# 東方BIG野球 試合処理手順書

ソース: プロデューサーからの口頭説明（2026-02-19〜2026-02-23）
目的: IRCパーサー実装の基盤資料。パーサー・ステートマシン開発時に参照すること。

---

## 1. 試合開始プロトコル

試合開始から最初の打席までの手順。

### 1.1 試合ルールマクロ

- どちらかのプレイヤーが「試合ルール」マクロを実行
- sianボットが球場・DH制・日付・季節・開始時間を出力
- 球場設定の数字（1112/1314等）はUP表の種類に対応

### 1.2 ホーム/ビジター決定

- 両者が1d20を実行
- 数値が大きい方がホーム（後攻）
- 同値の場合はやり直し

### 1.3 予告先発

- 両者が先発予定投手を発表
- オープナー宣言もこのタイミングで行う（第二先発可能投手も同時に宣言）

### 1.4 天候チェック

- 天候チェックマクロを実行
- 球場タイプ別に雨天中止確率が異なる: ドーム球場（殆ど中止にならない） ＜ 人工芝 ＜ 天然芝

**雨天中止の場合:**
1. 中止された試合は順延。日程表の空き日程（「予備日」優先、交流戦期間は「交流戦予備日」）を使用
2. 両チームの日程を1試合分進行
3. 再度予告先発を行う（変更なしなら「スライド」宣言で省略可）
4. 天候チェックを再実行
5. 催行可能になるまでこのループを繰り返す

予備日使用の判断: 通常はプレイヤー判断。9月以降または予備日残り3日以下で運営判断に切替。

### 1.5 オーダー発表（ビジター先→ホーム後）

各チームが以下の手順で発表:

#### 1.5a 1,2軍入れ替え発表
- 試合前に入れ替えがある場合、先に発表

#### 1.5b メンバー発表
必須記載事項:
- チーム名
- 日程の進捗（チーム日程表の日程）
- 特例指定選手
- スターティングメンバー（打順・守備位置）
- ベンチ入り選手
- ベンチ外選手（「オフ」= 戦略的にベンチから外す選手）
- 投手の疲労状況（「全快」= 累積0イニング）
- 登板履歴（最低7日分）
- （中継）マーク = リリーフ契約の投手（記載義務あり）
- 借金特例があれば記載

**注意**: オーダー発表の書式はプレイヤーごとに個性が大きい。

#### 1.5c 負傷チェック
1. 1d20を実行
2. 2以下が出た場合 → チームメンバーの誰かが負傷
3. 1d(1軍登録メンバー数)を実行 → メンバー発表時の出現順で上から数えて特定
4. 特定された選手に対し負傷判定プロトコル（→9.1節）を実行:
   - 指定選手判定 → 特徴判定 → 怪我レベルマクロで日数判定
   - **試合前の場合、怪我レベルは+1補正**（元が6の場合は6のまま）

#### 1.5d 怪我発生時の対応
- オーダーの変更が可能
- 出場選手登録の変更が可能

### 1.6 試合開始
- ビジター→ホームの順で処理完了後、挨拶をして試合開始

## 2. 1打席の完全処理フロー

### 2.1 打席間の手順（次打席の準備）

以下の順序で進行する:

1. **攻撃側: 結果発表 + 次打席の作戦発表**
   - 前打席の結果、走者状態、アウトカウントを表示（必須情報）
   - 得点状況は**得点が発生した場合のみ**記載（変化なしなら省略）
   - 次の打者と作戦を宣言（ヒッティングの場合は作戦省略が多い）
   - バント・エンドラン・盗塁はこのタイミングで宣言
   - 得点あり例: 「右本② 2-0 １死」（方向+結果+得点+スコア+アウト）
   - 得点なし例: 「遊ゴ　１死」（結果+アウト）
   - 走者がいる場合は走者状況も記載（例: 「１死１，３塁」）
   - **書式はプレイヤーの個性が出やすい**

2. **守備側: 守備位置宣言**
   - 前進守備等の宣言（毎打席必要、宣言なし=通常守備）
   - 投手・守備交代があればここで宣言（イニング中の場合）

3. **攻撃側: 作戦変更（任意）**
   - 守備位置を見て作戦を変更できる

4. **守備側: 投球マクロ実行**
   - sianボットがP列（投球番号1〜5）を判定

### 2.2 打撃処理

5. **攻撃側: 打撃マクロ実行**
   - 投球番号に対応するマクロを起動
   - sianボットが打撃結果コードを出力

6. **結果の3分岐**:
   - **通常の打撃結果** → 2.3へ
   - **レンジチェック** → 2.4へ
   - **UP表** → 2.5へ

### 2.4 レンジチェック処理

**打撃結果が「レンジ」の場合のみ発生する。** 通常の打撃結果やUP表からはレンジチェックに進まない。

1. 結果コードのポジション番号から該当野手を特定
2. その野手のカードに記載された守備力の数値を参照
   - 例: 二塁手の守備能力「4A」→ 数値=4、エラーランク=A
3. **内野レンジN** または **外野レンジN** のマクロを実行（Nは守備力数値）
4. sianボットが結果を出力
5. 結果は新たな打撃結果コードとして出力される
6. → 2.3（通常の打撃結果処理）に合流

**レンジチェック固有の結果**:
- LD（ライナー）: 打者アウト+先頭走者アウト（例外あり）
- 内野レンジがH/2Hになった場合: 対応する外野手が処理（1B→RF, 3B→LF, P/2B/SS→CF）
- Pのレンジ結果が2Hの場合はHとして扱う

### 2.5 UP表処理

> ※ エラーチェックはUP表処理の結果としてのみ発生する。レンジチェックの結果からエラーチェックへの遷移は発生しない。

出目20（一部例外選手は出目1）で発生。2diceで4桁コードを取得し、UP表を参照。

**主要パターン**:

| 種別 | 処理 |
|------|------|
| エラーチェック | 該当野手のエラーランクで判定（→2.5.1） |
| 死球 | DBと同じ処理 |
| 特徴 | 打者or投手の特徴が発動（→2.5.2） |
| 暴投 | 走者1〜2進塁 + 三振振り逃げの可能性 + 同打者で打席やり直し |
| 捕逸 | 走者1進塁 + 三振振り逃げの可能性 + 同打者で打席やり直し |

**暴投・捕逸の詳細**:
- 処理後、同じ打者の打席を最初からやり直す（投球から再開）
- バント作戦だった場合はスリーバント補正（-5）がかかる
- **WPしない特殊能力持ち投手**: 暴投時に走者進塁なし、そのまま振り直し（捕逸は通常通り進塁）

#### 2.5.2 特徴処理（UP表から発動）

UP表で「投手特徴確認」「打者特徴確認」を引くと、該当選手のカード記載の特徴が発動する。
- 野手が登板中に投手特徴確認 → **ホームラン**
- 投手が打席で打者特徴確認 → **三振**

**結果置換型**（打席結果をその特徴の結果に差し替える）:

| 特徴 | 発動側 | 処理 |
|------|--------|------|
| ホームラン | 投手 | HR処理（ソロ〜満塁、走者状況に応じた得点） |
| 三振 | 野手 | K処理（打者アウト） |
| DB | 投手/野手 | 死球処理（打者1塁へ）。この死球では怪我・危険球退場は発生しない（警告試合を除く） |
| 内野安打 | 野手 | 打者1塁へ。**1塁走者がいれば2塁フォースアウト**。**内野前進守備時は3塁走者がホームでアウト** |

**走者進塁型**（走者を進塁させ、投球からやり直し）:

| 特徴 | 発動側 | 処理 |
|------|--------|------|
| WP | 投手 | 暴投。全走者1進塁、投球やり直し。走者なしなら投球やり直しのみ |
| ボーク | 投手 | 全走者1進塁、投球やり直し。走者なしなら投球やり直しのみ |

**振り直し型**（指定列で打席をやり直す。ステートマシンには直接影響せず、パーサー側で解決済みの結果コードが来る）:

| 特徴 | 発動側 | 処理 |
|------|--------|------|
| 対右 | 投手 | 打者が右打者なら列5、左打者なら列1で振り直し |
| 対左 | 投手 | 打者が右打者なら列1、左打者なら列5で振り直し |
| 対右 | 野手 | 相手が右投手なら列1、左投手なら列5で振り直し |
| 対左 | 野手 | 相手が右投手なら列5、左投手なら列1で振り直し |
| 1C5 | 野手 | 走者なし時は列1、走者あり時は列5で振り直し |
| チャンス | 野手 | 走者なし時は列5、走者あり時は列1で振り直し |
| （代打） | 野手 | 代打時は列1で振り直し。代打でない時は他の特徴を使用 |

**特殊能力**（他の判定に補正を与える）:

| 能力 | 対象 | 処理 |
|------|------|------|
| WPしない | 投手 | あらゆる暴投を防ぐ。UP表の暴投は投球やり直し。捕逸は防げない |
| ボークしない | 投手 | ボークを防ぐ。UP表のボークは投球やり直し |
| クイック○ | 投手 | 盗塁st値出目+2、捕手肩+2（盗塁されにくい） |
| クイック✕ | 投手 | 盗塁st値出目-2、捕手肩-2（盗塁されやすい） |
| 風神少女 | 野手 | 走塁表出目-2。ただし出目20は補正なしでアウト確定 |

※ 負傷系特徴（けが、けが**日、DBけが**日、DB乱闘、精神疲労）は低頻度のため後日整理。

#### 試合中の降雨処理（UP表）

UP表から発生する試合中断イベント。**ドーム球場では何も起こらず、投球からやり直し。**

| UP表コード | 処理 |
|-----------|------|
| 1001〜1002 | 雨が降り出す。屋外球場で**4回以前ならノーゲーム**。5回以降なら試合成立扱いで雨天コールド。成立要件・成績処理・結果処理はNPBルール準拠 |
| 1003〜1004 | 雨が降り出す。屋外球場で試合成立済み（5回以上）なら即雨天コールド |

**ノーゲーム時の処理**:
- 試合記録（打撃成績等）は無効になるが、**負傷状況・投手の疲労状況の計算は行われる**
- 日程の再割当てが必要（試合開始プロトコル 1.4の雨天中止と同様の順延処理）

その他のUP表結果（詳細は後日整理）: 牽制アウト、ボーク、打撃妨害、守備妨害、等

#### 2.5.1 エラーチェック

**UP表の結果としてのみ発生する。** レンジチェックからは発生しない。

該当野手のエラーランク（S/A/B/C/D/E）で判定。エラーランクS=チェック不要で確定アウト。

**内野エラー（3パターン）**:

| 結果 | 処理 |
|------|------|
| 処理成功 | G#fと同じ処理 |
| ファンブル | 打者・走者1進塁 |
| 悪送球 | 打者・走者2進塁 |

**外野エラー（3パターン）**:

| 結果 | 処理 |
|------|------|
| 処理成功 | F#と同じ処理 |
| 落球 | 打者・走者1進塁（2死では走者のみ2進塁） |
| ワンヒットワンエラー | 打者・走者2進塁（打者に安打記録、自責点計算に注意） |

### 2.5.3 暴投追加処理（通称「4-19」）

通常の暴投（UP表経由）の発生頻度が低すぎるため追加されたバランス調整ルール。

**発生条件**:
- 無死or1死
- 走者が存在する
- 打撃結果がK（三振）

**基本ルール（4-19）**:
- 投球番号4・出目19のKで発生
- まず暴投が発生（走者1進塁、投手に暴投記録）→ その後打者は三振扱い

**特殊能力「暴投追加」持ち投手**:
- 投球番号5・出目19でも同様に発生（5-19）
- ただし打者の投球番号5が全Kの場合は発生しない（マクロを振らず即三振処理のため）

**WPしない投手**: 暴投なし、普通に三振（UP表の暴投と同じ例外）

### 2.6 走塁表判断

打撃結果処理後、積極的な走塁が可能な条件を満たしていれば:

1. 攻撃側が選択: 「回す」（走塁表実行）or「止まる」（省略して次へ）
2. 「回す」の場合: 任意で走塁表マクロ起動 → 1d20 → 走塁表と照合
3. **判定**: 状況に対応する走塁表を選択し、出目が閾値**以下ならセーフ**、**超えたらアウト**。値が0なら実行不可
4. 複数走者が対象の場合は**先頭走者のみ**判定
5. ほぼ全てのパターンで**3塁→本塁**の追加進塁（得点判定）

走塁表の詳細（閾値・補正値等）は `wiki/baserunning_table.md` を参照。

**走塁表対象の条件**（基本: 「a」なしの結果で、走力×外野手T値→1d20判定）:
- H#（外野安打）: 積極走塁可（**ただしH7の1塁走者は積極走塁不可**）
- H#a（長め安打）: 確定2進塁（H7aは1塁走者のみ確定1進塁+走塁表）
- 2H#（二塁打）: 1塁走者が積極走塁可
- F#（外野フライ）: 3塁走者が積極走塁可、F#aなら確定生還
- F8a/F9a: 2塁走者→3塁の積極走塁も可（例外的パターン）
- IH#（内野安打）: 積極走塁**不可**

**2死時の有利判定**: ライナーバック不要のため走者の判定が有利。ただし犠牲フライはベースタッチ必要のため適用外。

処理完了 → 2.1に戻り次打席へ。3アウトなら → 3（イニング終了）へ。

## 3. イニング終了処理

- 3アウトで「チェンジ」
- 結果発表に加えて**イニング終了宣言** + **終了イニング** + **得点状況**を表示（必須）
- 攻守交替して相手側のイニング開始宣言へ

### イニング間の処理

イニング間に以下が行われる場合がある:
- 投手交代・守備交代（前イニング終了後〜次の投球前、イニング開始宣言の前後どちらでもOK）
- 攻撃側が続投を確認（続投宣言→最低1打者完了まで交代不可）

## 4. 選手交代プロトコル

### 4.1 共通手順

1. **タイム**（任意）: 交代したい側がタイムをかける。誤ってゲームが進行することを防ぐ
2. タイム〜作戦決定までに振られたダイスは**無効**

### 4.2 代打・代走

- 交代宣言と次の作戦をセットで宣言
- 作戦が明示されない場合はヒッティング扱い
- 守備側が作戦を確認する慣例がある
- 例: 「代打：魔理沙　１塁代走：妖夢」「霊夢→代打：咲夜　送りバント」
- 結果宣言やイニング開始宣言と同時に代打宣言される場合がある

### 4.3 投手・守備交代

- **イニング中**: 攻撃側の作戦発表後に行う
- **イニング間**: 前イニング終了後〜次の投球前（攻撃側イニング開始宣言の前後どちらでもOK）
- 宣言方法は個性が出やすい（例: 「投手交代：霊夢　守備交代：センター文」）

### 4.4 続投確認ルール

- イニング間に攻撃側が相手投手の続投を確認できる
- 続投を宣言した場合、最低1打者との対戦を完了するまで交代不可（負傷・退場を除く）

### 4.5 リエントリー制限

- リエントリー制度なし（一度交代した選手は再出場不可）
- 大谷ルール例外: 投手を降りてもDHとして継続は「交代」ではない

## 5. 盗塁手順

### 5.1 スタート判定（st）

1. 攻撃側が盗塁を宣言（作戦発表タイミング）
2. マクロ `{背番号}-st{塁}` を実行
3. sianボットが1d20を出力
4. スタート補正: 二盗=左投手のみ-2、三盗=-4（左右問わず）、ホームスチール=-8（左右問わず）
5. 出目20 → 牽制発生

**牽制処理**:
- 二盗・三盗: 攻撃側1d20で10以下=帰塁セーフ、11以上=アウト
- ホームスチール: ダイス判定なし、自動牽制アウト

### 5.2 エンド判定（ed）

1. スタート成功後、マクロ `{背番号}-ed{塁}` を実行
2. sianボットが1d20を出力
3. ed補正: 捕手のT値（肩力）+ 投手のクイック特殊能力

> ※ 本盗（ホームへの盗塁）: ed値に-8の補正を加えて判定を行う。捕手のT値（肩力）は適用されない（詳細: `wiki/steal.md`）。

**確定結果**（補正に関係なく）:
- 二盗: 出目3以下=確定成功、20=確定失敗
- 三盗: 出目2以下=確定成功、20=確定失敗
- 本盗: 出目1=確定成功、20=確定失敗

### 5.3 重盗（ダブルスチール）

1. 先頭走者のスタート判定(st)を実行
2. 成功後、**守備側が送球先を選択**
3. 選択された走者のエンド判定(ed)を実行
4. 送球先選択前にedダイスを振った場合は原則やり直し（両者合意で続行する場合もある）

### 5.4 盗塁の制約

- **成功後**: 次の塁に進むまで再度盗塁指示不可（ゲームバランス調整）
- **スタート失敗後**: 打撃結果で進塁するまで再盗塁不可

### 5.5 盗塁後の行動

**スタートを切った場合**（成功・失敗問わず）:
- 処理後に**両者が作戦変更・選手交代が可能**
- バントを選択できる（**2ストライク扱い**で-5修正のスリーバント。ファール→三振）

**スタートを切れなかった場合**:
- 作戦変更・選手交代は**不可**
- バントは選択可能（同じくスリーバント扱い）

## 6. バント手順

### 6.1 バントマクロ体系

| マクロ | 条件 |
|--------|------|
| `#-b-{塁}` | 通常バント / スクイズ |
| `#-s-{塁}` | 内野前進守備時 |
| `#-b3-{塁}` | スリーバント（通常守備） |
| `#-s3-{塁}` | スリーバント + 前進守備 |

- `{塁}` は先頭走者がいる塁を指定
- セーフティバントの条件を満たす場合は1塁の結果を参照

### 6.2 バント判定計算

バント判定値 = 打者のバント値(1〜10) + 1d20出目 - スリーバント補正(-5) - 前進守備補正(スクイズ時に-5)

### 6.3 バント表の特徴

- 通常のレンジチェック表は使用しない
- バント表詳細ページに条件が直接記載される
- FC表（フィルダースチョイス）がバント処理で使用される場合がある

## 7. エンドラン手順

### 7.1 宣言条件

- **走者1塁** または **走者1,3塁** の場合のみ宣言可能
- 盗塁処理後はエンドラン宣言不可（スタートしなかった場合も含む）

### 7.2 処理の流れ

1. 攻撃側が作戦発表タイミングで「エンドラン」を宣言
2. 投球→打撃の通常処理を実行
3. 打撃結果に応じた変化処理（→7.3）を適用

### 7.3 エンドラン時の打撃結果変化

走者がスタートを切っているため、打撃結果が以下のように変化する。

#### ゴロ系の変化

| 通常結果 | エンドラン時 | 理由 |
|----------|-------------|------|
| G#f | **G#aに変更** | 走者がスタート済みでフォースプレイ困難 |
| G#a | **変化なし** | 元からアドバンス |
| G4D, G6D | **変化なし** | 二遊間の併殺は成立 |
| その他のG#D (G1D, G3D, G5D等) | **G#aに変更** | 走者が動いているため併殺困難 |

#### ヒット系の変化

| 通常結果 | エンドラン時 | 理由 |
|----------|-------------|------|
| H# | **H#aに変更** | 走者がスタート済みで余分に進塁 |
| H7a | 通常は1塁走者1進塁+走塁表 → **1塁走者が3塁に確定進塁** | スタート済み |
| 2H#, 2H#a | **H#aに変更** | 走者の進塁分で結果格下げ |
| 3H# | **H#aに変更** | 同上 |
| HR# | **2H#aに変更** | 同上 |

#### 特殊結果の変化

| 通常結果 | エンドラン時 | 理由 |
|----------|-------------|------|
| H8a | **G6Dに変更** | センター方向ゴロ→走者スタートで2塁カバーに入った遊撃手が処理→併殺 |
| 内野レンジでアウト | **LD（ライナー）扱い** | スタートした走者が戻れない |

#### その他の結果

| 結果 | エンドラン時の処理 |
|------|-------------------|
| BB | **ファール扱いでやり直し（エンドラン状態は解除される）** |
| DB | エンドラン無関係に通常出塁 |
| PO | 通常と同じ（打者アウト・走者そのまま） |

### 7.4 エンドラン時の三振（K）処理

1. 打者は三振でアウト
2. 走者はスタートを切っているため、**エンドラン失敗後の盗塁判定**を行う
3. **ed判定のみ**実施（st判定なし）
4. 成功値 = **走力×2**（ただし個人の盗塁成功値(ed値)が低い場合はそちらを採用）

### 7.5 エンドラン状態の解除

以下の場合、エンドラン状態は解除され、その打席が終わるまで**再宣言不可**:

- **UP表発生時**: 通常状態に戻してからUP表の処理を行う。暴投・捕逸でやり直しになっても再宣言できない
- **BB発生時**: ファール扱いでやり直しになるが、エンドラン状態は解除される

## 8. 前進守備宣言

### 8.1 ルール

- **内野前進**: 無死or1死で走者3塁の場合のみ宣言可
- **外野前進**: 常に宣言可（走者なしでも可だが意味なし）
- 内野前進と外野前進は別ルール、それぞれ個別宣言が必要
- 「前進」だけ = 内野前進。「内外野前進」= 両方
- **毎打席宣言必須**。宣言なし = 通常守備（サヨナラ場面でも省略不可）

### 8.2 タイミング

- 攻撃側の結果発表+作戦発表の後に宣言
- 攻撃側は守備位置を見て作戦を変更できる

### 8.3 内野前進時の補正

- **ゴロ打球への補正**: 打撃結果表に内野前進時の処理が記載されている（ホーム送球等）
- **バント（スクイズ）への補正**: マクロが変更（`#-b-` → `#-s-`、`#-b3-` → `#-s3-`）＋ バント判定値に-5補正

### 8.4 外野前進時の補正

外野前進は打球が抜けた際の被害が大きくなる代わりに、走塁表の判定で有利になる。

#### 打球結果への補正

| 通常結果 | 外野前進時 | 理由 |
|----------|-----------|------|
| F#a | **2H#aに変更** | 前進しているため飛球が頭上を越えて長打に |
| 2H# | **2H#aに変更** | 前進しているため打球処理が遅れ走者一掃 |

#### 走力による追加補正

- 打者の走力が**3以上**の場合、上記補正を含む**全ての二塁打（2H#, 2H#a）が3H#（三塁打）に格上げ**
- 連鎖例: F#a → 2H#a → 3H#、2H# → 2H#a → 3H#

#### 走塁表への補正

- 積極走塁の走塁表判定に補正がかかる（外野が前進しているため）

#### 例外

- **外野レンジ捕球後のF#a**: レンジチェックで捕球が成立した場合は補正なし、そのままF#aとして処理

## 9. 試合中の負傷・退場

### 9.1 試合中の負傷判定プロトコル

カード特徴やUP表から負傷イベントが発生した場合、以下の手順で処理する:

1. **指定選手判定**: 特例起用の選手かどうかを確認
2. **特徴判定**: カードの特徴に基づく処理を実行
3. **怪我レベルマクロで日数判定**: 怪我表を参照して休場日数を決定
4. **交代処理**: 必要なタイミングで選手交代（打席途中なら打席完了後、イニング間ならそのタイミング）

※基本的な流れは試合前の負傷チェック（1.5c節）と同じ。

### 9.2 危険球退場（UP表 2016・2017）

1. 死球が発生 → 打者はDB処理で出塁
2. **投手は危険球退場**（即座に交代が必要）
3. **打者は負傷判定**を受ける（→9.1のプロトコル）
   - ただし怪我レベルマクロの段階で、**怪我レベル3以下の選手は怪我レベル4に補正**

### 9.3 警告試合中のDB退場

警告試合中に死球が発生した場合:
- **投手が退場**（危険球退場と同じ扱い）
- **打者の負傷判定はなし**

※警告試合の発動条件: DB乱闘後に危険球を受けた側がDB発生、または互いにDB3個以上 → 以後DB発生で投手無条件退場

### 9.4 三振抗議退場（UP表 2019・出目10以下）

- 打者が三振に抗議して**打者が退場**
- 三振扱い（打者アウト）

### 9.5 乱闘退場（UP表 2019・出目11以上）

頭部をかすめる死球による乱闘。最も複雑な退場パターン。

1. 死球が発生 → 打者はDB処理で出塁
2. **打者が乱闘を起こし退場**
3. **投手が暴行を受ける** → **怪我レベル6**で怪我チェック
4. 3で投手が**負傷しなかった場合** → 投手が追加で1d20、**11〜20なら危険球退場**（10以下なら続投可能）
5. **打者** → **怪我レベル5**で怪我チェック、出た日数が**謹慎処分**（出目「20」は5日扱い）

## 11. 試合終了条件

### 11.1 通常終了
- 9回表終了時点でホーム側リードなら**9回裏は行わず**試合終了
- 9回裏終了時点でビジター側リードなら試合終了

### 11.2 サヨナラ
- 9回裏以降、ホーム側が勝ち越した時点で試合即終了
- 結果発表+得点状況に試合終了宣言が加わる
- そのイニングの残り打席は行われない

### 11.3 延長戦
- 9回終了時点で同点の場合、延長戦に突入
- 通常のイニングと同様に進行
- 各イニング裏終了時に点差があれば試合終了
- **12回打ち切りで引き分け**（タイブレークなし）

## パーサー実装上の注意点

### 書式の揺れ
- 結果発表、オーダー発表、選手交代宣言はプレイヤーごとに個性が大きい
- 必須情報（得点・走者・アウト等）は含まれるが書式は不定

### タイミングの揺れ
- 代打宣言が結果宣言やイニング開始宣言と同一行に含まれる場合がある
- 投手・守備交代のイニング間の出現位置が揺れる
- 「回す」宣言や走塁表マクロ起動が省略される場合がある

### ダイスの無効化
- タイム〜作戦決定間のダイスは無効
- 重盗時の送球先選択前のedダイスは原則無効（両者合意で有効化する場合あり）

### 暴投・捕逸のやり直し
- 暴投・捕逸後は同じ打者の打席が再開される
- パーサーでは1打席が「中断→再開」として2回の投球サイクルを含む場合がある

### 投手疲労の特殊ルール（ステートマシン関連）
- **5イニング以上無失点ルール**: 得点を取られるまで疲労しない（疲労状態の投手も解除）。ただし10回からはランナーを出すと疲労、11回からは最初から疲労状態
- **疲労Pの次イニング猶予**: 疲労P回数を投げた次のイニングでは、敬遠以外のランナーを出すまで疲労しない
- 詳細は `game-flow.md §「投手疲労（＊マーク）」` および `card-structure.md §「疲労P」` を参照

### 試合途中終了（ステートマシン関連）
- **雨天コールド/ノーゲーム**: UP表1001〜1004で発生。試合成立（5回完了）の有無で処理が分岐
- **サヨナラ**: 9回裏以降、ホーム側勝ち越し時点で即終了。イニング途中でも終了
- ステートマシンは毎打席後に終了条件をチェックする必要がある
