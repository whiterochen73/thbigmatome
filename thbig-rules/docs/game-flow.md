# 試合進行の仕組み

東方BIG野球の1試合がどのように進行するかを解説します。IRCチャット上でダイスボットを介して行われるテキストベースの野球ゲームです。

## 登場人物

| 役割 | 説明 |
|------|------|
| 先攻プレイヤー | 先攻チームの監督。打撃コマンドを入力する |
| 後攻プレイヤー | 後攻チームの監督。打撃コマンドを入力する |
| ダイスボット（sian） | IRCチャット上のボット。コマンドに応じてダイスを振り、結果を出力する |

## 試合前の手順

1. **試合ルール確認** — ダイスボットが球場設定・DH制の有無・日付・季節・開始時間を表示する
2. **先攻後攻の決定** — 両プレイヤーがそれぞれ1d20（20面ダイス）を振り、出目が大きい方が後攻になる
3. **先発投手の宣言** — 先攻→後攻の順に先発投手を宣言する
4. **天候チェック** — ダイスボットが天候（晴天/雨天）を決定する
5. **オーダー発表** — 両チームがスタメン・控え選手・投手陣・登板履歴をチャットに提示する
6. **怪我チェック** — 両チームが1d20を振る。2以下が出ると選手が怪我をする
7. **試合開始** — 先攻の打順から試合が始まる

## 1打席の流れ

### 基本フロー

```
守備側: 投球コマンド（投手背番号-p）
   ↓
ダイスボット: 投球ナンバーを出力（1d20）
   ↓
攻撃側: 打撃コマンド（打者背番号-投球ナンバー）
   ↓
ダイスボット: 打撃結果コードを出力
   ↓
必要に応じて追加判定（UP表・走塁表・エラーチェックなど）
   ↓
攻撃側が状況を宣言（例:「遊ゴ 1死」）
```

### 結果コードの種類

打撃結果はアルファベットと数字の組み合わせで表現されます。

#### アウト系

| コード | 意味 |
|--------|------|
| K | 三振 |
| G + 守備番号 | ゴロアウト（例: G6 = 遊撃手ゴロ） |
| F + 守備番号 | フライアウト（例: F9 = 右翼フライ） |
| GD | ゲッツー（ダブルプレー） |
| LD | ライナーアウト |
| PO | ポップアウト（内野フライ） |

#### 安打系

| コード | 意味 |
|--------|------|
| H + 打球方向 | 単打（例: H6 = 遊撃手方向へのヒット） |
| IH | 内野安打 |
| 2H | 二塁打 |
| 3H | 三塁打 |
| HR | 本塁打 |

#### 特殊系

| コード | 意味 |
|--------|------|
| BB | 四球 |
| DB | 死球 |

コードに `a` が付く場合は追加進塁を意味します（例: `H6a` = 単打で走者が1つ余分に進む、`F9a` = 犠牲フライ）。

IRCログでは全角文字（`Ｈ９`など）が使われることがあります。

## イニングの進行

### 表と裏

1イニングは「表」（先攻チームが攻撃）と「裏」（後攻チームが攻撃）で構成されます。

- 3アウトで攻守交代（チェンジ）
- 表が終わったら裏へ、裏が終わったら次のイニングの表へ
- 通常は9イニングで試合終了

### 試合終了条件

- 9回裏終了時点でスコアに差がある場合、試合終了
- 9回裏に後攻チームがリードしている場合、サヨナラで即終了
- 同点の場合は延長イニングに入る

## 特殊イベント

### UP表（好プレイ珍プレイ表）

通常の打席結果とは別に、特殊イベントが発生することがあります。2つの20面ダイス（d20）を振り、その組み合わせで4桁のコード（例: `0703`）を生成して、UP表から対応するイベントを参照します。

UP表のイベントには以下のカテゴリがあります。

| カテゴリ | 内容 |
|---------|------|
| エラーチェック | 特定の守備手にエラー判定が発生する |
| 雨天中断 | 試合が一時中断する |
| 死球 | 打者が死球を受ける |
| 投手/打者特性 | 投手・打者固有のイベント |
| 走者依存イベント | 走者の有無によって結果が変わる（暴投・ボーク・捕逸など） |
| 球場設定依存 | 球場の設定によって結果が異なる特殊プレイ |
| 負傷・特殊UP | 選手の負傷や特殊プレイ |

走者依存イベントは特に重要です。たとえば暴投（ワイルドピッチ）は、走者がいる場合は走者が進塁しますが、走者がいない場合は投手のエラーチェックになります。

### 盗塁

攻撃側は打席コマンドの代わりに盗塁コマンドを入力できます。選手カードの盗塁開始値・盗塁完了値とダイスの出目で成否が判定されます。

### エンドラン（ヒットエンドラン）

走者がスタートを切った状態で打撃を行う作戦です。三振になった場合は走者の盗塁成否を別途判定します。

### バント

打者がバントを試みる場合、通常とは異なるテーブルで結果が判定されます。選手カードのバント能力値が成功率に影響します。

### 投手交代・代打・代走

イニング中に投手交代・代打・代走が行われることがあります。これらは打席結果とは独立した宣言として処理されます。

先発投手が5イニング未満で交代した場合は「KO（ノックアウト）」扱いとなります。

## IRCログの構造

### チャンネル

本番環境ではIRCチャンネルが複数並行しており（`#thc_BIGbaseball` ～ `#thc_BIGbaseball4`）、チャンネルごとに独立した試合が進行します。

### ログの読み方

1行が1つのアクション（コマンド入力またはボット出力）に対応します。

```
[チャンネル] [ニックネーム] [メッセージ]
```

ダイスボットの出力行には `result:` が含まれ、その後に結果コードが続きます。

### 保留処理

一部のイベントは1打席で完結せず、追加のダイス判定が必要になります。

| 状況 | 追加判定 |
|------|---------|
| 走塁表が必要な結果 | 走塁表（baserunning table）でアウト/進塁を判定 |
| 暴投・捕逸 | 同じ打者が再度打席に立つ |
| エンドランで三振 | 走者の盗塁成否を判定 |

これらの保留処理は次の打席に入る前に解決されます。
