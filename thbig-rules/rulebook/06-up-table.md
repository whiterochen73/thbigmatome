# 第6章: UP表（好プレイ珍プレイ表）

**品質基準**: エミュレーターが本章の処理を実装できるレベル
**出典**: wiki/highlight_table.md（Wikiスクレイプ生データ）

---

## 6.0 概要

UP表（好プレイ珍プレイ表）は、打撃結果コードが `UP` だったときに参照する特殊イベント表。
2つのd20ダイスで生成した4桁コードによって試合中の特殊事象（エラー・怪我・ワイルドピッチ・ホームラン等）を決定する。

- **発動タイミング**: 通常打席処理フローで打撃結果が `UP` と判定された打席
- **発動回数**: 1打席あたり1回（追加d20が必要なコード除く）
- **キャンセル**: エンドラン宣言中にUP表が出た場合、エンドランは無効になる

---

## 6.1 4桁コード生成

```
INPUT: なし（2回のd20ロールのみ）

STEP 1: 1つ目のd20ロール → 千の位・百の位を決定
  出目01 → "01"（前2桁）
  出目02 → "02"
  ...
  出目20 → "20"

STEP 2: 2つ目のd20ロール → 十の位・一の位を決定
  出目01 → "01"（後2桁）
  出目02 → "02"
  ...
  出目20 → "20"

STEP 3: 連結してコード生成
  例: 1つ目=19、2つ目=07 → コード "1907"

OUTPUT: 4桁コード（0101〜2020の範囲）
```

**コード範囲**: 理論上0101〜2020（d20の01〜20 × 01〜20）

---

## 6.2 球場設定と特殊分岐（コード1911〜1914）

コード1911〜1914は **球場設定** によって参照するコードが変わる。

| 球場設定 | 参照するコード | 方向 |
|---------|--------------|------|
| 1112 | 1911〜1912 | ライト方向（両翼） |
| 1314 | 1913〜1914 | センター方向 |

```
BALL_PARK_BRANCH（コード1911-1914到達時）:
  IF venue.setting == "1112":
    参照コード = 1911 または 1912（ライト方向）
  ELIF venue.setting == "1314":
    参照コード = 1913 または 1914（センター方向）
```

### 6.3.1 エラーチェック（0101〜0920）

コードに対応する守備手のエラーチェックを実施する。

```
INPUT: code (0101〜0920)
OUTPUT: error_check_result

守備手マッピング:
  0101〜0204 → 投手（P）のエラーチェック
  0205〜0220 → 捕手（C）のエラーチェック
  0301〜0320 → 一塁手（1B）のエラーチェック
  0401〜0504 → 二塁手（2B）のエラーチェック
  0505〜0620 → 三塁手（3B）のエラーチェック
  0701〜0816 → 遊撃手（SS）のエラーチェック
  0817〜0904 → 左翼手（LF）のエラーチェック
  0905〜0912 → 中堅手（CF）のエラーチェック
  0913〜0920 → 右翼手（RF）のエラーチェック

→ エラーチェック判定方法: wiki/error_check_table.md 参照
```

### 6.3.2 天候・死球（1001〜1020）

| コード | 結果 |
|--------|------|
| 1001〜1002 | 雨が降り出す。屋外球場なら「5回未満→ノーゲーム / 成立→雨天コールド」 |
| 1003〜1004 | 雨が降り出す。屋外球場で試合成立済みなら雨天コールド |
| 1005〜1020 | 死球（打者に投球が当たる → 打者1塁） |

```
RAIN_EVENT_FLOW:
  IF venue == dome:
    → 何も起こらず投球やり直し（at_bat_redo = true）
  ELIF inning <= 4:
    → ノーゲーム（records_valid=false, fatigue_valid=true）
  ELSE:
    → 雨天コールド（records_valid=true）

HIT_BY_PITCH:
  打者は1塁に出塁
  走者は1塁フォースなら押し出し
```

### 6.3.5 特殊プレー（1901〜1920）

| コード | 結果 |
|--------|------|
| 1901〜1910 | 捕手レンジチェック。レンジ0〜1: ホームラン / レンジ2: 打者は1で振り直し / レンジ3以上: レンジチェック→アウトならキャッチャーフライ、それ以外はファール+投球やり直し |
| 1911〜1912 | 球場確認（ライト方向）。F9a→ライトフライ（3塁走者生還・2塁走者3塁へ）/ それ以外→ホームラン |
| 1913〜1914 | 球場確認（センター方向）。F8a→センターフライ（3塁走者生還・2塁走者3塁へ）/ それ以外→ホームラン |
| 1915 | ワイルドピッチ。走者2進塁 + 打者振り逃げ（1進塁）。ただし2死以外で1塁走者あり→三振のみ |
| 1916 | ファーストライナー（打者アウト）。全走者が飛び出してアウト |
| 1917 | 打球が2塁ベース直撃しG4f。1塁走者なし + 打者走力4以上 or 内野前進守備 → IH4 |
| 1918 | G3f。一塁手レンジ3以下→2H9 |
| 1919 | 強烈なピッチャーゴロG1a。投手は負傷→怪我チェック発動 |
| 1920 | キャッチャーベンチ激突。打者アウト・走者そのまま。捕手は怪我チェック発動 |

### 6.3.7 追加d20が必要なコード（2018〜2020）

```
INPUT: code (2018〜2020)

STEP 1: 追加d20ロール（1〜20）

STEP 2: コード別分岐
  code == 2018:
    1〜15 → 守備妨害（打者アウト）
    16〜20 → 打撃妨害（打者出塁）
  
  code == 2019:
    1〜10 → 三振（打者はアウト後に退場）
    11〜20 → 頭部かすり死球 → 乱闘退場フロー:
      1. 死球で打者出塁（通常の死球と同じ）
      2. 打者が乱闘を起こし退場
      3. 投手にけが特徴6（最軽傷）でチェック → 負傷なし確認
      4. 投手：1d20で11以上 → 危険球退場（投手退場）
      5. 打者にけが特徴5でチェック → 出た日数が謹慎処分（出目20は5日）
      ※ けが特徴方向: 1=最重傷、6=最軽傷（数字が小さいほど重い）
  
  code == 2020:
    1〜10 → 先頭走者が隠し球でアウト（走者なしなら三振）
    11〜20 → ランニングホームラン
```

## 6.4 UP表処理完全フロー

```
UP_TABLE_FLOW:
  INPUT: 
    game_state（走者状況・アウト数・球場設定・守備陣形等）
  
  STEP 1: 2回のd20ロール → 4桁コード生成
  
  STEP 2: コード範囲判定
    0101〜0920 → エラーチェック（§6.3.1）
    1001〜1020 → 天候・死球（§6.3.2）
    1101〜1520 → 特徴確認（§6.3.3）
    1601〜1820 → 走者関連イベント（§6.3.4）
    1901〜1920 → 特殊プレー（§6.3.5）
    2001〜2017 → 野手負傷（§6.3.6）
    2018〜2020 → 追加d20判定（§6.3.7）
  
  STEP 3: コード1911〜1914の場合
    球場設定を確認して分岐（§6.2）
  
  STEP 4: コード1601〜1820の場合
    has_any_runner を確認して分岐（§6.3.4）
  
  STEP 5: 結果に基づきゲーム状態を更新
    得点 / アウト数 / 走者位置 / 怪我フラグ 等
  
  OUTPUT: UP_TABLE_OUTCOME（通常の打席処理に合流）
```
