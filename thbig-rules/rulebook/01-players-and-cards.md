# 第1章: 選手とカード

**状態**: 実装済み
**品質基準**: エミュレーターがカードデータを読み込み、能力値を参照して試合処理を行えるレベル

---

## 概要

東方BIG野球の選手はすべて「カード」として表現される。カードには打撃結果表・投球データ・守備値・走力・バント値・盗塁力・けが特徴・スタミナなどの属性が含まれ、試合中のあらゆる判定の基礎となる。

本章では、カードの種別・構造・各能力値の定義と範囲・バリエーションの概念を定義する。

---

## 1.1 カード種別

選手カードは2種別に大別される。

| 種別 | 識別名 | 説明 |
|------|--------|------|
| 投手カード | `pitcher` | 投球データ（P列）を持ち、投手として登板できる |
| 野手カード | `batter` | 投球データを持たず、野手として出場する |

- 同一選手が投手カードと野手カードの両方を持つことがある（二刀流選手）
- 種別はカードの属性であり、同一選手でもカードごとに異なる
- → 外の世界枠のバランスルール（第2章）で投手/野手の判定に使用

## 1.2 カードの一意性制約

```
UNIQUE(card_set_id, player_id, card_type, variant)
```

- 同一カードセット内で、同一選手の同種カードは原則1枚
- `variant`（後述）が異なれば、同一選手・同一種別で複数枚存在可能
- 異なるカードセット間では同一選手の複数カードを自由に持てる

## 1.3 カードセット

カードは「カードセット」という単位で管理される。

| カードセット | 作品 | 背番号の特徴 |
|-------------|------|------------|
| 2025THBIG | 東方Project | 数値（例: 1, 29, 165） |
| hachinai | 八月のシンデレラナイン | 数値。DB登録時にP/F接頭辞を付与 |
| PM2026 | GLADIATS等 | 3桁・0始まりあり（例: 001, 004） |
| tamayomi | 球詠 | 数値。DB登録時にT接頭辞を付与 |

背番号は文字列型で保持する（接頭辞やゼロ埋めがあるため）。

## 1.4 バリエーション（variant）

一部のカードセット（PM2026等）では、同一選手・同一種別のカードが複数存在する。

```yaml
# 例: PM2026の初瀬麻里安（投手）
- variant: null     # 通常版
- variant: "湘南"   # 湘南バージョン
- variant: "町田"   # 町田バージョン
- variant: "UR"     # URバージョン
```

- `variant`が`null`の通常カードは従来の1枚制限に従う
- `variant`が非nullのカードは、variantの値で区別される

## 1.5 投球カード（P列）

投手カードのみが持つ。出目1〜20に対応する投球ナンバー1〜5の配列。

### 投球ナンバーの意味

| 番号 | 意味 | 打者への影響 | 投手にとっての評価 |
|------|------|------------|-----------------|
| 1 | 甘い球 | 長打されやすい | 悪い |
| 2 | やや甘い球 | そこそこ打てる | やや悪い |
| 3 | 打たせて取る球 | アウトを取りやすい | 良い |
| 4 | ボール球 | 四球の可能性 | やや悪い |
| 5 | 三振球 | ほぼ三振 | 最強 |

### P列の配置ルール

- 基本的に出目が小さいほど小さい投球番号が並ぶ（1→2→3→4→5の順）
- スラッシュ（`/`）やP分岐は末尾側に配置される傾向がある

### 疲労マーカー（`*`）

投球番号に`*`が付いた場合、その投手が疲労状態のとき1段階劣化する。

```
劣化ルール:
  2* → 1（やや甘い球 → 甘い球）
  3* → 2（打たせて取る球 → やや甘い球）
  5* → 4（三振球 → ボール球）
```

- `*`は単独の投球番号2・3・5にのみ付加される（1・4には付かない）
- 同一番号内では`*`付きを上（小さい出目側）に配置
- 合計1〜4個（疲労耐性により異なる）
- 例外: 東風谷早苗は`*`が6個（通常上限超え）

### スラッシュ分岐（`/`）— 対左右打者

```
記法: 投球A/投球B
意味: /の左 = 対左打者時、/の右 = 対右打者時
例:   1/3 → 左打者には投球1（甘い球）、右打者には投球3（打たせて取る球）
```

同腕対決（右投手vs右打者、左投手vs左打者）では投手有利に設定される傾向がある。

### P分岐 — 走者有無条件

```
記法: 投球AP投球B
意味: Pの左 = 走者なし時、Pの右 = 走者あり時
例:   1P3 → 走者なしなら投球1、走者ありなら投球3
```

### 例外選手

| 選手 | 特徴 | 処理 |
|------|------|------|
| 稀神サグメ | 投球番号が上下逆（5→1の順） | 出目20→1方向で読み取り |
| 東風谷早苗 | `*`が6個（通常上限4を超過） | `*`上限チェックで例外扱い |

## 1.6 打撃カード（打撃結果表）

### 構造

5列（投球ナンバー1〜5）× 20行（出目1〜20）= 100マスの打撃結果表。

```
マス数:
  野手:           5×20 = 100マス
  投手（通常）:   4×20 =  80マス（投球5の列は省略、全てK扱い）
  投手（5|P併記）: 5×20 = 100マス（投球5に打撃結果が存在する投手）
```

### 打撃結果の優先順位

カードのマスは以下の優先順位で上位出目から埋まる傾向がある。

```
優先順（高い順）:
  1. HR  — 本塁打
  2. 3H  — 三塁打
  3. 2H  — 二塁打
  4. H   — 単打
  5. IH  — 内野安打
  6. レンジ — 守備判定マス
  7. G   — ゴロ（守備番号付き: G6等）
  8. F   — フライ（守備番号付き: F9等）
  9. PO  — ポップフライ
 10. BB  — 四球
 11. DB  — 死球
 12. K   — 三振
```

### 投球ナンバー別の傾向

| 投球ナンバー | 傾向 | 評価基準 |
|-------------|------|---------|
| 1（甘い球） | 長打（HR）が集中 | — |
| 2（やや甘い球） | HR・3H・2H・Hが混在 | **長打力**: 投球2のHR数（0=低, 1=標準, 2=高, 3+=極高） |
| 3（打たせて取る球） | 単打(H)が多い | **巧打力**: 投球3のH有無・対応度 |
| 4（ボール球） | BB（四球）が主、DB（死球）もここ | — |
| 5（三振球） | 通常は全てK | **三振耐性**: 投球5の構成（全K=低, 混在=高） |

### K以降全Kルール

```
ルール: Kが単独で出現した最初の出目以降、出目19まで全てK
例:     投球3で出目12がK → 出目13〜19も全てK
例外:   出目20はUP表参照（下記参照）
```

### 出目20のルール

```
通常:   出目20 = UP表（好プレイ珍プレイ表）を参照して特殊イベント判定
例外:   投球5が全マスK（確定三振選手）の場合 → 出目20もK（UP表参照しない）
```

### レンジマスの配置

打撃結果表内のレンジマスは、投球ナンバーごとに固定的に分布する。

| 投球ナンバー | レンジマス個数 |
|-------------|-------------|
| 1 | 2個 |
| 2 | 3個 |
| 3 | 3個 |

### スラッシュ分岐（`/`）— 対左右投手

```
記法: 打撃結果A/打撃結果B
意味: /の左 = 対左投手時、/の右 = 対右投手時
例:   K/H9 → 左投手には三振、右投手にはライト前ヒット
```

同腕対決（右投手vs右打者、左投手vs左打者）では投手有利に設定される傾向がある。

### スイッチヒッターの扱い

両打の打者は投手のスラッシュ投球を受けた場合、打者に有利な方が自動適用される。

### 例外選手

| 選手 | 特徴 | 処理 |
|------|------|------|
| 鬼人正邪 | 打撃結果の順番が逆転（出目1にUPが来る等） | 出目20→1方向で読み取りイメージ |

## 1.7 守備値

各ポジションごとに設定される守備能力。

### 守備値の構成

| 要素 | 説明 | 値域 | 用途 |
|------|------|------|------|
| レンジ値 | 守備範囲の広さ | 0〜5（0=狭い, 5=広い） | レンジチェック判定の基準値 |
| エラーランク | 守備の確実性 | S/A/B/C/D/E | エラーチェック時のDP/ファンブル/悪送球判定 |
| T値（外野手） | 肩の強さ | S/A/B/C（文字） | 走塁判定・盗塁判定で使用 |
| T値（捕手） | 肩の強さ | ±数値 | 走塁判定・盗塁判定で使用 |

### エラーランクS

- ランクSは通常ルールブックに記載のない特殊ランク
- 多数試合出場かつエラーなしの選手に付与
- 効果: エラーチェック不要で確定アウト（e判定省略）

### T値の適用範囲

| ポジション | T値 | 備考 |
|-----------|-----|------|
| 捕手 | 数値±表記（例: T: +1） | 盗塁阻止能力 |
| 外野手 | ランク（例: T: B） | 走塁判定への補正 |
| 内野手 | なし | 実装上は0扱い |

### 守備ポジション表記

```
基本:     P(1)/C(2)/1B(3)/2B(4)/3B(5)/SS(6)/LF(7)/CF(8)/RF(9)
外野統合: OF = LF/CF/RF共通守備値
外野個別: LR(両翼)/LC(レフト+センター)/CR(センター+ライト)
```

- OF（統合）vs 個別（LF/CF/RF/LR/LC/CR）は排他的
- 複数ポジションの守備値を持つ選手がいる（コンバート可能）

## 1.8 走力

走塁能力。盗塁判定・エンドラン・走塁表の判定に使用。

| 値 | 意味 | 走塁表への影響 |
|----|------|-------------|
| 1 | 遅い | 成功値が厳しい |
| 2〜4 | 通常 | 平均的な判定 |
| 5 | 速い | 成功値が有利 |
| 6 | 特殊 | 「風神少女」能力発動時のみ到達 |

走力6は射命丸文の特殊能力「風神少女」発動時専用。走塁表のダイス目に−2補正（ただし出目20は補正なし、自動アウト）。

**値域**: 1〜5（通常）、6（特殊能力発動時のみ）
→ `game_rules.yaml`: `thbig_baseball.player_card.speed` (min: 1, max: 5)

## 1.9 バント値

バント能力。1〜10の整数値。

```
バント判定値 = 打者のバント値 + 1d20出目
             − スリーバント補正（−5）
             − 前進守備補正（−5）
```

| 値 | 意味 |
|----|------|
| 1〜3 | 苦手（成功率低） |
| 4〜7 | 通常 |
| 8〜10 | 名手（成功率高。ただし前進守備+スリーバントで−10されると厳しい） |

→ `game_rules.yaml`: `thbig_baseball.player_card.bunt` (min: 1, max: 10)

## 1.10 盗塁力

盗塁の成否を決定する2つの値。

| 属性 | 値域 | 説明 |
|------|------|------|
| 盗塁開始値（steal_start） | 1〜22 | ダイス値との比較で盗塁を試みるか決定 |
| 盗塁完了値（steal_end） | 1〜22 | steal_startと組み合わせて盗塁成功判定 |

→ `game_rules.yaml`: `thbig_baseball.player_card.steal_start/steal_end` (min: 1, max: 22)

## 1.11 けが特徴（負傷率）

選手が負傷する確率と重傷度を決定する値。

| 値 | 意味 | 説明 |
|----|------|------|
| 0 | テーブル不要 | けが判定表を参照しない |
| 1 | 最重傷 | 長期離脱（最大60日） |
| 2〜5 | 中間 | 値が大きいほど軽傷 |
| 6 | 最軽傷 | ほぼ出場可能 |
| 7 | 判定免除 | けが判定自体が発生しない（フルイニング出場可） |

→ `game_rules.yaml`: `thbig_baseball.player_card.injury_rate` (min: 0, max: 7)

**正式名称**: 「けが特徴」（けがレベルとも呼ばれる）

### けが判定表（概要）

けが特徴1〜6の選手が負傷した場合、1d20を振り、けが特徴レベルとの交差セルから休場日数を取得する。

- `*` = 怪我なし
- `→` = けがレベル−1して振り直し
- `0` = その試合のみ出場不可
- 数値 = 休場日数

→ 詳細は wiki/injury_table.md を参照

### 特徴けが日数ルール（＊ルール）

カードの特徴欄にけが免除や日数指定が記載される場合がある。

```
処理順（上から順に消化）:
  ＊           → 免除回数（怪我せず判定終了）
  日数のみ     → その日数分負傷
  日数（するごと） → 毎回その日数の怪我（消化されず繰り返し）
  （するごと）のみ → けがレベルで怪我チェック
  全エントリ消化後 → カード記載のけが特徴レベルで通常判定に戻る
```

## 1.12 スタミナ（疲労P）

投手の投球可能回数。投手区分の判定にも使用。

### 投手区分の判定

| 値 | 区分 |
|----|------|
| 0〜3 | リリーフ（中継ぎ） |
| 4〜9 | 先発 |

→ `game_rules.yaml`: `thbig_baseball.player_card.stamina` (starter: 4-9, relief: 0-3)

### スラッシュ形式

```
記法: 先発値/リリーフ値
例:   7/2 → 先発時スタミナ7、リリーフ時スタミナ2
```

先発とリリーフで異なるスタミナを持つ投手（スラッシュ選手）。

### 特殊修飾

| 修飾 | 意味 | 例 |
|------|------|-----|
| R | リリーフ専任。他に先発可能投手がいれば先発不可 | 1R |
| C | クローザー。9回以降のみ登板可能 | 3C |

### スラッシュ選手の優遇

スラッシュ選手（先発/リリーフ兼任）は、先発→リリーフ転向時に通常より1日短い間隔で回転可能。
→ 詳細は投手休養ルール（第9章）を参照

### KO（ノックアウト）

先発投手が5イニング未満で交代して敗戦した場合。
→ `game_rules.yaml`: `thbig_baseball.game.ko_innings_threshold` (value: 5)

## 1.13 特徴・特殊能力

カードの特徴欄に記載される、UP表で発動する能力。

### 投手特徴

UP表で「投手特徴確認」を引くと発動する。

| 特徴 | 効果 |
|------|------|
| WP | 暴投。走者1進塁して再投球。走者なし時は投球から振り直し |
| DB | 死球（怪我・危険球退場は発生しない） |
| ボーク | 走者1進塁して再投球。走者なし時は投球から振り直し |
| ホームラン | ホームランを被弾 |
| けが | 投手が自分のけがレベルで怪我チェック |
| けが＊＊日 | その日数分怪我して降板。投手交代必要 |
| 対右 | 右打者なら5、左打者なら1で打つ |
| 対左 | 右打者なら1、左打者なら5で打つ |

投手特徴を持たない野手が投手として登板中にこの枠を引いた場合: **ホームラン**

### 野手特徴

UP表で「打者特徴確認」を引くと発動する。

| 特徴 | 効果 |
|------|------|
| DB | 死球（怪我・危険球退場は発生しない） |
| DBけが＊＊日 | 死球で日数分怪我。代走が必要 |
| DB乱闘 | 死球で乱闘、打者退場。投手はけがレベル6で怪我チェック |
| けが＊＊日 | 日数分負傷。代打が必要 |
| 1C5 | 走者なし時は1、走者あり時は5で振り直し |
| チャンス | 走者なし時は5、走者あり時は1で振り直し |
| 内野安打 | 内野安打。1塁走者ありなら2塁フォースアウト。前進守備時は3塁走者がホームアウト |
| 対右 | 右投手なら1、左投手なら5で振り直し |
| 対左 | 右投手なら5、左投手なら1で振り直し |
| （代打） | 代打時は1で振り直し。代打でない時は他の特徴を使用 |

野手特徴を持たない投手が打席に立ってこの枠を引いた場合: **三振**

### 特殊能力

| 能力 | 対象 | 効果 |
|------|------|------|
| WPしない | 投手 | あらゆる暴投を防ぐ。UP表の暴投は投球から振り直し。パスボールは防げない |
| ボークしない | 投手 | ボークを防ぐ。UP表のボークは投球から振り直し |
| 精神疲労 | 投手 | 通常の疲労に加え、失点時点で疲労状態になる |
| クイック○ | 投手 | 盗塁されにくい。st値出目+2、捕手肩+2 |
| クイック✕ | 投手 | 盗塁されやすい。st値出目−2、捕手肩−2 |
| 風神少女 | 野手 | 走塁表の出目を−2補正。出目20は補正なし（アウト確定） |

**注意**: カード上は「特徴」と「特殊能力」が混在して記載される場合がある。特殊能力は特徴より上に記載される傾向がある。

## 1.14 打順

試合は9人打線で行われる。DH（指名打者）制採用時も打順は1番〜9番の9枠。
→ `game_rules.yaml`: `thbig_baseball.game.batting_order` (min: 1, max: 9)
