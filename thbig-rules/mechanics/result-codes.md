# 結果コード体系 定義書

**Layer B 概念定義書 — 雛形**
作成: 2026-04-04 ashigaru4 (subtask_907a)
実装: `thbig-irc-parser/src/thbig_irc_parser/result_code.py`

---

## 1. 概要

結果コード（Result Code）は、1打席の結果をIRCログから抽出した文字列表現。
GSM（Game State Machine）がこのコードを解釈して試合状態（走者・アウト・得点）を更新する。

全角文字（IRC由来）は正規化後に解釈される。

---

## 2. コード一覧

### 2.1 アウト系

| コード | 意味 | 例 |
|--------|------|-----|
| `G{N}D` / `GD` | ゲッツー（ダブルプレー） | `G5D` |
| `G{N}a` / `Ga` | ゴロ（進塁打） | `G6a` |
| `G{N}f` / `Gf` | ゴロ（フォースアウト） | `G4f` |
| `F{N}` / `F` | 浅い外野フライ | `F9`（右飛） |
| `F{N}a` / `Fa` | 深い外野フライ（犠牲フライ） | `F9a` |
| `LD` | ライナーアウト | — |
| `K` | 三振 | — |
| `PO` | ポップアウト（内野フライ） | — |

### 2.2 安打系

| コード | 意味 |
|--------|------|
| `H{N}` | 単打（打球方向N） |
| `H{N}a` | 単打（追加進塁あり） |
| `IH{N}` / `IH` | 内野安打 |
| `2H{N}` / `2H` | 二塁打 |
| `2H{N}a` | 二塁打（追加進塁あり） |
| `3H{N}` / `3H` | 三塁打 |
| `HR{N}` | 本塁打 |

### 2.3 特殊系

| コード | 意味 |
|--------|------|
| `BB` | 四球（打者1塁へ） |
| `DB` / `死球` | 死球（打者1塁へ） |

### 2.4 GSM内部で生成されるコード（IRCログ由来ではない）

UP表解決後に `gsm_result_code` として上書きされるケースがある（例: `K`、`LD`、`G4f`、`G3f`）。

---

## 3. 全角文字の正規化

IRCログは全角英数字を含むことがある。Parserは送信前に `str.maketrans` で半角に変換する。

例: `Ｈ９` → `H9`、`Ｋ` → `K`

---

## 4. エイリアス

| 入力 | 正規コード | 理由 |
|------|----------|------|
| `死球` | `DB` | IRC日本語表記の吸収 |
| `エラー(1進塁)` | `IH` | 1進塁エラーを内野安打として近似処理 |

---

## 5. 走者状況キー

| キー | 状態 |
|------|------|
| `empty` | 走者なし |
| `1` | 1塁のみ |
| `2` | 2塁のみ |
| `3` | 3塁のみ |
| `1,2` | 1・2塁 |
| `1,3` | 1・3塁 |
| `2,3` | 2・3塁 |
| `full` | 満塁 |

GSM内部では走者状況キーで打席結果ハンドラーをルックアップする。

---

## 6. AtBatOutcome フィールド

打席処理後のGSM出力:

| フィールド | 型 | 意味 |
|------------|-----|------|
| `runs_scored` | int | この打席で入った得点数 |
| `is_change` | bool | 3アウトでチェンジ |
| `is_game_over` | bool | 試合終了 |
| `pending_baserunning` | bool | 走塁表（baserunning table）が必要 |
| `at_bat_redo` | bool | WP/PBで同打者再打席 |
| `pending_steal` | bool | エンドランK時の1塁走者盗塁判定が必要 |
| `pending_steal_base` | str | 盗塁判定が必要な走者の塁 |
