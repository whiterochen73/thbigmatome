## 0. PDF別概要（解析ソース）

カードデータは複数のPDFソースから解析される。各PDFの特徴を以下にまとめる。

| PDF名 | 作品 | 背番号の特徴 | 備考 |
|-------|------|------------|------|
| 2025THBIG | 東方Project | 接頭辞なし（例: 1, 29, 165） | 東方キャラのみ |
| hachinai | 八月のシンデレラナイン | 接頭辞なし（例: 1〜10） | 同一選手に投手版(pitcher)と野手版(batter)が存在。接頭辞P/FはDB登録時に付与 |
| PM2026 | GLADIATS等 | 3桁・0始まりあり（例: 001, 004, 125） | 接頭辞なし。横綱カード3名はスコープ外 |
| tamayomi | 球詠 | 接頭辞なし（例: 1, 6, 26） | 川崎稜に1年版・2年版の2バージョンあり。接頭辞TはDB登録時に付与 |

> **注意**: 背番号接頭辞（P/F/T等）はパーサー段階では付与しない。DB登録時に付与する。

## 1. 投球カード（P列）

### 投球番号の意味

| 番号 | 意味 | 評価 |
|------|------|------|
| 1 | 甘い球 | 悪い（長打されやすい） |
| 2 | 少し甘い球 | やや悪い |
| 3 | 打たせて取る球 | 良い（アウト取りやすい） |
| 4 | ボール球 | 悪い（四球の可能性） |
| 5 | 三振球 | 最強 |

### P列の並び順ルール

- 基本: 上から **1 → 2 → 3 → 4 → 5** の順に並ぶ
- 分岐出目（スラッシュ `/` や `P`）は、**上の投球番号の末尾**に配置
- `/` は `P` より上に配置

### 疲労マーカー（`*`）のルール

- `*` は単独の投球番号 **2・3・5 のみ**に付く（1・4には付かない）
- 同一番号内では **`*`付きを上**に配置
- 合計 **1〜4個**（疲労耐性依存）
- 疲労状態では `*` 付き投球番号が1段階劣化: `2*→1`、`3*→2`、`5*→4`

### スラッシュ形式（`/` 分岐）

**対左右打者分岐**:
- `「対左結果/対右結果」` の形式
- 例: `1/3` = 対左打者→投球1、対右打者→投球3
- 同腕対決は投手有利になるよう設定

**P分岐（P表記）**:
- 走者有無条件: 左=走者なし、右=走者あり

### 例外選手

| 選手 | 例外内容 |
|------|---------|
| 稀神サグメ | 投球番号が上下逆（5→1の順） |
| 東風谷早苗 | 高能力だが `*` が6個（通常上限4を超える） |

## 2. 打撃カード

### 出現優先順

打撃結果のカード内表記は以下の順で並ぶ:

```
HR# → 3H# → 2H# → H# → IH# → レンジ → G# → F# → PO → BB → DB → K
```

- `BB` / `DB` は原則として**投球番号4のみ**に出現

### 同一結果内の並び順

- **ポジション番号が若い順**（例: G3 → G4 → G5 → G6）
- ゴロのサブタイプ: **D → f → a** の順
  - 例: `G3D → G3f → G3a`
- `a` つき結果: ポジション別で**上側**（同ポジションならaありが先）

### スラッシュの配置ルール

- **有利な方の結果の最後**に配置
- `「対左結果/対右結果」` 形式は投球カードと共通

### レンジ目の分布

| 投球番号 | レンジ目の数 |
|---------|------------|
| 1 | 2つ |
| 2 | 3つ |
| 3 | 3つ |

### 例外選手

| 選手 | 例外内容 | 設計思想 |
|------|---------|---------|
| 鬼人正邪 | 打撃結果の順番が逆転、出目1にUPが来る | キャラの「ひっくり返す」能力の再現 |

※稀神サグメの投球逆転と同様の設計思想

## 3. 疲労P（スタミナ）

### fatigue_p の全パターン

カードに記載される疲労P（`fatigue_p`）の値と、DBフィールドへのマッピングは以下の通り。

| 記載パターン | 例 | 区分 | DBフィールド |
|------------|-----|------|-------------|
| 単独 0〜3 | `2` | リリーフ投手 | `relief_stamina = 2`、`starter_stamina = 3`（先発可能だが3扱い） |
| 単独 4〜9 | `7` | 先発投手 | `starter_stamina = 7`、`relief_stamina = null`（リリーフ時0扱い） |
| スラッシュ形式 | `6/1` | 先発・リリーフ両用 | `starter_stamina = 6`、`relief_stamina = 1` |
| R付き | `1R`, `0R` | リリーフ専任 | `is_relief_only = true`、`relief_stamina = 数値部分` |
| `null`（記載なし） | — | 野手 | fatigue_p不要（野手カード） |

### 先発とリリーフの範囲

| 区分 | 疲労P値 |
|------|--------|
| リリーフ | 0〜3 |
| 先発 | 4〜9 |

数字のみ記載の場合は範囲で先発/リリーフを判別可能。

### スラッシュ形式

- `「先発スタミナ/リリーフ疲労P」` の形式
- 例: `7/2` = 先発時7、リリーフ時2

### R付き投手の制約

- 疲労Pに `R` 付き（例: `1R`, `0R`）はリリーフ専任
- 他に先発可能投手がいる場合、先発不可（オープナールール除く）

### Rなしリリーフ投手の先発利用

- リリーフ値0〜3の `R` なし投手は**疲労P3扱いで先発可能**

### 先発投手のリリーフ運用

- リリーフ値未記載の先発投手は**疲労P0でリリーフ可**

### 「明記」と「未記載」の違い

| 記載パターン | 例 | 意味 | 投手休養での回復速度 |
|------------|-----|------|-------------------|
| 明記（スラッシュ形式） | `6/0` | 先発6、リリーフ0 | 早い（先発・リリーフ両方の経験値） |
| 未記載（数字のみ） | `6` | 先発6、リリーフ未記載（P0） | 遅い |

> **実装上の注意**: データモデルでは「リリーフ値明記の有無」を区別する必要あり

## 4. 守備値の特殊ポジション

### 外野統合ポジション

カードには「OF」（外野統合）のほか、以下の複合ポジションが存在する。

| フィールド名 | 対応ポジション | 意味 |
|------------|-------------|------|
| `OF` | 外野全体統合 | LF/CF/RF を統合して1つの守備値を持つ |
| `LR` | 両翼（Left+Right） | レフトとライトを統合した守備値 |
| `CR` | 中右（Center+Right） | センターとライトを統合した守備値 |
| `LC` | 左中（Left+Center） | レフトとセンターを統合した守備値 |

### OF統合と個別の排他関係

- `OF`（統合）と個別ポジション（`LF`/`CF`/`RF`）は**排他的**
  - `OF` がある場合、`LF`/`CF`/`RF` は存在しない
  - `LR`/`CR`/`LC` は `OF` より細分化されたパターン
- パーサーは記載されているキーをそのまま格納する

### 内野・捕手の守備値

内野（1B/2B/3B/SS）と捕手（C）は個別フィールドで管理。複合表記はない。

## 5. 選手評価基準

### 長打力（投球2のHR数で評価）

| 投球2のHR | 推定長打力 |
|---------|----------|
| 2-1 HR（2の出目で1マスHR） | 24本級 |
| 2-2 HR（2の出目で2マスHR） | 38本級 |
| 2-3 HR（2の出目で3マスHR） | 55本級 |

### 巧打力（投球3の内容で評価）

| 投球3の内容 | 評価 |
|-----------|------|
| 3-1確定ヒット + 3-2スラッシュ | 打率.330+・タイトル級 |
| 3-1レンジ | 打率.200以下 |

### 三振耐性（投球5の内容で評価）

| 投球5の内容 | 評価 |
|-----------|------|
| 全K（全マス三振） | 三振率30%+（ただし長打力が高い選手も存在する） |

### 投手の打撃評価

- 基本: 投球5は**確定三振扱い**（打撃マクロ不要、即三振処理）
- 例外1: 投球5の打撃結果が用意されている投手（稀）
- 例外2: 投球5は確定三振でも他番号で打力が高い投手

## 6. 特徴（traits）の主要パターン

### バイオリズム

- 二十四節気ベースの特徴。該当季節中に発動する
- カード上では複数行にまたがる形式で記載（日付範囲 + 効果 + 別条件の効果等）
- **現在の出力**: 複数の文字列に分割して `traits` 配列に格納（3分割が多い）
- **将来対応**: 専用フィールド（`player_card_biorhythms` テーブル）へ昇格予定

### 1C5パターン（走者有無での打撃変化）

- カードに「1C5」と記載される特徴。**走者なし時は投球1で振り直し、走者あり時は投球5（確定三振）**
- 2025THBIG解析で**32選手が確認済み**
- `abilities.batting_skills` に格納

### 専属捕手（catcher_pitching_style）

- 特定の捕手とペアで投球特徴が変化する（通常より有利な結果になりやすい）
- カードの特徴欄に捕手名が記載される（例: `exclusive_catcher: レティ`）
- DBフィールド: `catcher_pitching_style_id`（専属捕手時の投球特徴ID）

### 走者あり/なし分岐（pinch_pitching_style）

- 投手が走者の有無で投球特徴を切り替えるパターン
- カード上は「(有走者)」「(無走者)」等の条件付き特徴として記載
- DBフィールド: `pinch_pitching_style_id`（走者あり時の投球特徴ID）

### 特殊ケース（injury_level 関連）

| 選手 | 特徴内容 | 実装上の扱い |
|------|---------|------------|
| 渡里ニナ | けが100日 | `injury_level` 不要（特徴で日数固定） |
| 鈴仙（今季絶望） | 今季絶望 | `injury_level` 不要（特徴で固定） |

## 7. スコープ外カード

以下のカードはパーサーの対象外、またはDB登録対象外とする。

### 横綱カード（PM2026、3名）

- PM2026 に含まれる3名の横綱カード
- 公式戦（Lペナ等）のスコープ外のため、DB登録対象外
- パーサーで解析はするが、インポートスクリプトでスキップ

### UR折り返しレイアウト（PM2026、13名+1名）

- PM2026 の一部カードはURレアリティで、複数行が折り返された特殊レイアウト
- 標準の行抽出処理では正しく解析できない（複数行マージ処理が必要）
- **現時点でスコープ外**: パーサー改修後に対応予定
- 対象: 13名 + 追加1名（計14名）

## まとめ: カード設計の原則

| 項目 | 基本原則 |
|------|---------|
| P列並び | 1→2→3→4→5の昇順。スラッシュは末尾、`/`は`P`より上 |
| 疲労マーカー | 2・3・5のみ。同番号内は`*`あり優先。合計1〜4個 |
| 打撃カード出現順 | HR→3H→2H→H→IH→レンジ→G→F→PO→BB→DB→K |
| レンジ目分布 | 投球1に2つ、投球2・3に各3つ |
| 疲労P判別 | 4以上=先発、3以下=リリーフ。スラッシュで両用。Rつき=リリーフ専任。null=野手 |
| 外野守備 | OF統合 or LR/CR/LC 複合 or 個別(LF/CF/RF)。OF統合と個別は排他 |
| 例外処理 | 稀神サグメ（逆順投球）・鬼人正邪（逆順打撃）は別処理が必要 |
| スコープ外 | PM2026横綱カード（3名）・URレイアウト（14名）はDB登録対象外 |
