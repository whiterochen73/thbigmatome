# UP表（好プレイ珍プレイ表）概念定義書

作成: 2026-04-04 ashigaru4 (subtask_907a)
実装: `thbig-irc-parser/src/thbig_irc_parser/up_table.py`

## 1. 概要

UP表（好プレイ珍プレイ表）は、通常の打席結果（GSM結果コード）の外で特殊イベントを解決するための参照表。
2つのd20ダイスを振り、4桁コード（例: `0703`）を生成してイベント種別を決定する。

**出典**: [THBIGウィキ 好プレイ珍プレイ表](https://thbigbaseball.wiki.fc2.com/wiki/%E5%A5%BD%E3%83%97%E3%83%AC%E3%83%BC%E7%8F%8D%E3%83%97%E3%83%AC%E3%83%BC%E8%A1%A8)

---

## 2. コード生成ルール

| ダイス | 結果例 |
|--------|--------|
| d1（1〜20） | `07` → 0詰めして2桁 |
| d2（1〜20） | `03` → 0詰めして2桁 |
| 4桁コード | `d1 * 100 + d2` = `0703` |

範囲: `0101` 〜 `2020`

---

## 3. コード範囲とカテゴリ一覧

| コード範囲 | カテゴリ | 概要 |
|-----------|---------|------|
| 0101〜0920 | エラーチェック | 守備手（P/C/1B/2B/3B/SS/LF/CF/RF）のエラー判定へ |
| 1001〜1020 | 雨天中断 | 試合中断・流れ判定 |
| 1101〜1120 | 死球 | 打者死球、1塁へ進塁 |
| 1201〜1220 | 投手特性 | 投手特有イベント |
| 1301〜1320 | 打者特性 | 打者特有イベント |
| 1401〜1420 | 未使用/予備 | — |
| 1501〜1520 | 未使用/予備 | — |
| 1601〜1819 | 走者依存イベント | 走者あり/なしで結果が変わる（WP・ボーク・牽制アウト・PBなど） |
| 1901〜1920 | 球場設定依存 | 球場設定（1112/1314）によって結果が異なるスペシャルプレイ |
| 2001〜2020 | 負傷・特殊UP | 選手負傷、特殊プレイ |

---

## 4. 主要カテゴリの詳細

### 4.1 エラーチェック（0101〜0920）

コード値によって守備手が決まり、その守備手のエラー率でエラー発生を判定する。

| コード | 守備手 |
|--------|--------|
| 0101〜0204 | 投手 (P) |
| 0205〜0220 | 捕手 (C) |
| 0301〜0320 | 一塁手 (1B) |
| 0401〜0504 | 二塁手 (2B) |
| 0505〜0620 | 三塁手 (3B) |
| 0701〜0816 | 遊撃手 (SS) |
| 0817〜0904 | 左翼手 (LF) |
| 0905〜0912 | 中堅手 (CF) |
| 0913〜0920 | 右翼手 (RF) |

### 4.2 走者依存イベント（1601〜1819）

走者の有無で結果が反転する。

**一般原則**: 走者なし → 特定守備手のエラーチェック（投手ECが最多）

例:
- `1601`: 走者あり → 暴投（走者2進塁）/ 走者なし → 投手EC
- `1602〜1608`: 走者あり → 暴投（走者1進塁）/ 走者なし → 投手EC

### 4.3 球場設定依存（1901〜1920）

- 設定1112: `1901〜1910`
- 設定1314: `1911〜1920`
- `1901〜1902`: 両設定共通（捕手レンジチェック）

## 5. Parser実装との対応

```
UPResolution {
  category: str    # カテゴリ識別子
  fielder: str     # エラーチェック対象守備手 (nullable)
  details: str     # 人間向け説明文
  gsm_result_code: str  # GSMへの上書きコード (nullable)
}
```

`resolve_up_code(code_str, has_runners, stadium_setting)` でUPResolutionを取得し、
GSM状態機械（game_state.py）に渡す。
